# 重构计划模板（强制检查版）

> 文档类型：可执行重构计划  
> 状态规则：任务必须使用 `- [ ]` / `- [x]`  
> 停止规则：存在任意 `- [ ]` 时，不得宣布“完成/停止/归档”

---

## 0. 执行状态总览

- [ ] 完成现状盘点
- [ ] 完成目标架构定义
- [ ] 完成执行计划拆分
- [ ] 完成守卫与测试验收

---

- [ ] 目标 2

### 1.2 架构目标

- [ ] 目标 1

## 2. 现状问题（证据）

## 3. 执行计划（Phase）

### Phase-0：冻结与基线

- [ ] 冻结非重构改动
- [ ] 固化基线测试

### Phase-1：主链路改造

- [ ] 切换到唯一入口
- [ ] 删除旧并行入口

### Phase-2：验收与发布

- [ ] `go test ./...` 通过
- [ ] 架构守卫通过
- [ ] 文档同步完成

---

## 4. 删除清单（必须执行）

## 5. 完成定义（DoD）

- [ ] 无并行旧路径残留
- [ ] 所有测试通过
- [ ] 文档与代码一致
- [ ] 所有任务状态均为 `- [x]`
