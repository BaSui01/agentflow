# 中文重构计划治理

## 快速流程

1. 先运行格式检查：
   - `python scripts/refactor_plan_guard.py lint`
2. 再看进度报告：
   - `python scripts/refactor_plan_guard.py report`
3. 每次准备“结束/收尾/归档”前必须跑门禁：
   - `python scripts/refactor_plan_guard.py gate`
4. `gate` 未通过时，继续推进计划项，禁止以“已完成”收尾。

## 硬规则

1. 计划文件必须包含任务状态行：`- [ ]` 或 `- [x]`。
2. 计划文件必须包含三个章节组：
   - `执行状态总览`
   - `执行计划` 或 `Phase`
   - `完成定义` 或 `DoD`
3. 未全部打勾（仍有 `[ ]`）时，不得输出“完成/停止/归档”结论。
4. 重构执行遵循单轨替换，不允许兼容双实现。

## 计划维护动作

1. 新建计划时，直接复用模板：
   - `./references/重构计划模板.md`
2. 审核计划时：
   - 先标冲突（状态与代码不一致、路径错误、守卫声明错误）
   - 再给修订项（具体到文件与行）
3. 推进计划时：
   - 只把完成项从 `[ ]` 改为 `[x]`
   - 不删除未完成项
   - 每阶段至少保留一条可验证证据（测试/脚本/代码路径）

## 资源

### scripts/

1. `run_guard.ps1`：在项目根目录调用 `scripts/refactor_plan_guard.py`。

### references/

1. `重构计划模板.md`：规定章节和 `[ ]` 清单格式。

## 输出要求

1. 输出必须包含：
   - 当前剩余 `[ ]` 数量
   - 未完成项清单
   - 下一步执行项（仅来自现有未完成任务）
2. 只有 `gate` 通过时，才允许输出“可停止/可收尾”。
