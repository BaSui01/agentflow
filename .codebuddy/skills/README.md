# AgentFlow Skills

基于 [mattpocock/skills](https://github.com/mattpocock/skills) 适配的 CodeBuddy 技能集。

## 分类

### Engineering — 日常编码核心

| 技能 | 描述 |
|------|------|
| [diagnose](engineering/diagnose.md) | 系统化 Bug 调试：复现 → 假设 → 插桩 → 修复 → 回归 |
| [grill-with-docs](engineering/grill-with-docs.md) | 深度访谈 + 文档产出，精炼领域术语，更新 CONTEXT.md 和 ADR |
| [improve-codebase-architecture](engineering/improve-codebase-architecture.md) | 发现架构改进机会，基于领域语言和 ADR 深化模块设计 |
| [prototype](engineering/prototype.md) | 快速构建可运行原型，验证设计方向 |
| [setup-agentflow-skills](engineering/setup-agentflow-skills.md) | 一次性初始化技能配置（Issue 追踪器、标签、领域文档） |
| [tdd](engineering/tdd.md) | 红-绿-重构 TDD 循环，垂直切片实现功能 |
| [to-issues](engineering/to-issues.md) | 将计划/PRD 拆解为独立的 GitHub Issue |
| [to-prd](engineering/to-prd.md) | 将当前对话合成 PRD 并发布为 Issue |
| [triage](engineering/triage.md) | Issue 分类状态机（Bug/Enhancement + 标签流转） |
| [zoom-out](engineering/zoom-out.md) | 从宏观角度解释不熟悉的代码 |

### Productivity — 通用工作流工具

| 技能 | 描述 |
|------|------|
| [caveman](productivity/caveman.md) | 极简沟通模式，节省约 75% token |
| [grill-me](productivity/grill-me.md) | 需求深度访谈，澄清所有决策分支 |
| [write-a-skill](productivity/write-a-skill.md) | 按照标准结构创建新的 CodeBuddy skill |

### Misc — 辅助工具

| 技能 | 描述 |
|------|------|
| [git-guardrails](misc/git-guardrails.md) | 阻止危险 git 命令的安全钩子 |
| [setup-pre-commit](misc/setup-pre-commit.md) | 配置预提交钩子（格式化 + 类型检查 + 测试） |
