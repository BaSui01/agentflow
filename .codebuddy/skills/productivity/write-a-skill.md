# Write a Skill — 创建 Skill

## 标准目录结构

```
skill-name/
├── SKILL.md           # 主指令（必需）
├── references/        # 参考文档（可选，超出100行时拆分为此）
├── scripts/           # 工具脚本（可选，确定性操作）
└── assets/            # 模板/资源文件（可选）
```

## SKILL.md 模板

```markdown
---
name: skill-name
description: 简短描述。使用 when [具体触发条件]
---

# 技能名称

## 快速开始
[最小可行示例]

## 工作流
[带检查清单的逐步流程]

## 高级功能
[链接到 references/ 等]
```

## 描述要求

- 长度：最多 1024 字符
- 人称：第三人称
- 结构：第一句说明能力，第二句以 "使用场景：" 结尾
- 目的：让 CodeBuddy 一眼判断何时加载该技能

## 何时添加脚本（scripts/）

- 操作是确定性的（验证、格式化等）
- 同样的代码会被重复生成
- 错误需要明确处理

## 何时拆分文件

- SKILL.md 超过 100 行
- 内容涉及不同领域
- 高级功能很少用到

## 创建后检查清单

- [ ] 描述包含触发条件
- [ ] SKILL.md 不超过 100 行
- [ ] 无时效性信息
- [ ] 术语一致
- [ ] 包含具体示例
- [ ] 不嵌套引用
