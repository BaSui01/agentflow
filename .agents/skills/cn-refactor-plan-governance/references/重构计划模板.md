# 重构计划模板（TDD 强制版）

> 文档类型：可执行重构计划  
> 状态规则：任务必须使用 `- [ ]` / `- [x]`  
> 测试规则：测试计划必须采用 TDD，先写失败测试，再用最小实现转绿，最后重构并执行回归验证  
> 停止规则：存在任意 `- [ ]`，或任一验证命令/通过标准未达成时，不得宣布“完成/停止/归档”

## 3. 测试策略（TDD）

- [ ] 先写失败测试并确认红灯（验证命令：`go test ./path/to/pkg -run TestName`; 通过标准：新增或修改的测试先失败，且失败原因与待解决问题直接对应）
- [ ] 采用最小实现让测试转绿（验证命令：`go test ./path/to/pkg -run TestName`; 通过标准：目标测试转绿，且未引入兼容分支、双实现或额外兜底）
- [ ] 完成重构并执行回归验证（验证命令：`go test ./path/to/pkg`; 通过标准：相关测试集全部通过，且旧实现/重复路径已删除）

---

## 4. 执行计划（Phase）

- [ ] 完成 Phase 执行项总览（验证命令：`python scripts/refactor_plan_guard.py report --target <计划文件名> --require-tdd --require-verifiable-completion`; 通过标准：进度报告与计划状态一致，未完成项仍保留）

### Phase-0：冻结与基线

- [ ] 冻结非重构改动（验证命令：`git status --short`; 通过标准：仅保留本次重构相关改动）
- [ ] 固化基线测试（验证命令：`go test ./path/to/pkg`; 通过标准：记录当前基线结果，后续回归命令与范围固定）
- [ ] 固化结构基线（验证命令：`rg --files path/to/target`、目录统计脚本或守卫测试；通过标准：已记录当前包/目录/入口数量与职责基线）

### Phase-1：职责收口与唯一入口

- [ ] 产出职责矩阵并确认唯一正式入口（验证命令：人工审阅计划文档；通过标准：每个目标文件/目录都已标记“保留 / 合并 / 下沉 / 删除”，且唯一入口明确）
- [ ] 切换到唯一入口（验证命令：`rg "旧入口|旧构造|旧路由" path/to/code`; 通过标准：主链路只剩一个有效入口）
- [ ] 删除旧并行入口（验证命令：`rg "旧入口A|旧入口B" path/to/code`; 通过标准：旧实现与并行入口都不可再命中）

### Phase-2：包合并 / 下沉 / 删除落地

- [ ] 合并重复职责到唯一归口（验证命令：`go test ./path/to/pkg -run TestName`; 通过标准：重复语义只剩一处正式实现）
- [ ] 下沉或拆分过大的包/文件（验证命令：`rg --files path/to/target`; 通过标准：目标职责已落到新目录/子包，旧位置不再保留并行实现）
- [ ] 删除旧 facade / helper / runtime / adapter / legacy 入口（验证命令：`rg "旧类型名|旧函数名|旧目录名" path/to/code`; 通过标准：生产代码和正式文档中不再命中旧实现）

### Phase-3：验收与发布

- [ ] 相关测试通过（验证命令：`go test ./path/to/pkg`; 通过标准：退出码为 0，且无 fail 输出）
- [ ] 架构守卫通过（验证命令：`go test ./agent -run PlanGuard`; 通过标准：相关守卫测试或脚本退出码为 0）
- [ ] 文档同步完成（验证命令：`python scripts/refactor_plan_guard.py lint --target <计划文件名>`; 通过标准：计划结构合法，文档描述与代码现状一致）
- [ ] README / ADR / 架构文档 / 守卫同步完成（验证命令：`rg "唯一入口|旧入口|旧目录名" README.md README_EN.md docs scripts`; 通过标准：正式文档和守卫口径与重构后结构一致）

- [ ] 删除过期文档表述 / 守卫口径 / 目录说明（验证命令：`rg "旧架构描述|旧预算值|旧目录名" README.md README_EN.md docs scripts`; 通过标准：0 命中）

---

## 6. 完成定义（DoD / 严格完成标准）

- [ ] 无并行旧路径残留（验证命令：`rg "旧入口A|旧入口B|旧实现" path/to/code`; 通过标准：未命中任何应删除路径）
- [ ] 职责矩阵已闭环（验证命令：人工核对计划中的职责矩阵；通过标准：所有对象都已有最终归口，没有“待定/暂存”项）
- [ ] 唯一正式入口已收口（验证命令：`rg "Builder|Factory|Registry|runtime entry" README.md README_EN.md docs path/to/code`; 通过标准：正式文档与生产代码只保留一个正式入口口径）
- [ ] 新增与相关回归测试全部通过（验证命令：`go test ./path/to/pkg`; 通过标准：新增测试、受影响测试、守卫测试全部为绿色）
- [ ] 文档与代码一致（验证命令：`python scripts/refactor_plan_guard.py report --target <计划文件名>`; 通过标准：计划状态、证据路径、实际代码变更保持一致）
- [ ] 所有任务状态均为 `- [x]`（验证命令：`python scripts/refactor_plan_guard.py gate --target <计划文件名>`; 通过标准：退出码为 0，且允许停止/收尾）
