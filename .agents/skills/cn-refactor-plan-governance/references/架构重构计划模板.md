# 架构重构计划模板（包合并 / 唯一入口 / 目录治理专用）

> 文档类型：可执行架构重构计划  
> 适用场景：包拆分、包合并、重复功能统一、唯一入口收口、根目录治理、守卫预算调整、旧实现删除  
> 状态规则：任务必须使用 `- [ ]` / `- [x]`  
> 执行规则：单轨替换；不保留兼容分支；不保留双实现；先定义职责矩阵与唯一归口，再做迁移与删除  
> 测试规则：测试计划必须采用 TDD，先写失败测试，再用最小实现转绿，最后重构并执行回归验证  
> 停止规则：存在任意 `- [ ]`，或任一验证命令/通过标准未达成时，不得宣布“完成/停止/归档”
