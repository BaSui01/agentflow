# 核心开发规则

## 禁止行为

### 文档创建限制
- **严格禁止**创建总结文档、汇总文档、概述文档
- **严格禁止**创建 markdown 文件来记录工作过程或总结成果
- **严格禁止**创建不必要的文档文件
- 除非用户明确要求,否则不要生成任何形式的文档总结

### 代码最小化原则
- 只编写**绝对必要**的代码来解决问题
- 避免冗余实现和不直接贡献解决方案的代码
- 不要自动添加测试,除非用户明确要求
- 保持实现简洁,避免过度工程化

### 响应简洁性
- 完成工作后使用**最少的文字**表达结论
- 避免冗长的总结或详细的工作回顾
- 不要提供项目符号列表总结
- 说得少,做得多

## 工作流程

1. 理解用户需求
2. 用最少的代码实现
3. 验证功能
4. 用一两句话说明完成情况

## 记住

- 用户不需要文档噪音
- 用户需要的是工作代码,不是工作报告
- 简洁就是效率
