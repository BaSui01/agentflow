# AgentFlow Quick Start Guide

## Introduction

AgentFlow is a high-performance Go AI Agent framework providing unified LLM abstraction, intelligent routing, tool calling, and workflow orchestration.

Official entrypoint policy:

- Repository-level official entry: `sdk.New(opts).Build(ctx)`
- `agent/runtime.Builder` is the runtime entry for the `agent` submodule
- `agent.NewAgentBuilder`, `agent.CreateAgent`, and `agent.NewBaseAgent` remain available only for advanced extension or low-level customization

## Installation

```bash
go get github.com/BaSui01/agentflow
```

## Core Concepts

### 1. Provider

Provider is the unified abstraction interface for LLM services, supporting OpenAI, Claude, Gemini, DeepSeek, and more.

```go
import (
    "github.com/BaSui01/agentflow/llm/providers"
    "github.com/BaSui01/agentflow/llm/providers/openai"
)

provider := openai.NewOpenAIProvider(providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey:  os.Getenv("OPENAI_API_KEY"),
        BaseURL: "https://api.openai.com",
        Model:   "gpt-4o-mini",
    },
}, logger)
```

### 2. Agent

For normal integration, start from the repository-level SDK runtime:

```go
import (
    "context"

    "github.com/BaSui01/agentflow/sdk"
    "github.com/BaSui01/agentflow/types"
)

ctx := context.Background()

cfg := types.AgentConfig{
    Core: types.CoreConfig{
        ID:   "my-agent",
        Name: "Assistant",
        Type: "assistant",
    },
    LLM: types.LLMConfig{
        Model:       "gpt-4o-mini",
        MaxTokens:   4096,
        Temperature: 0.7,
    },
}

rt, err := sdk.New(sdk.Options{
    Logger: logger,
    LLM: &sdk.LLMOptions{
        Provider: provider,
    },
    Agent: &sdk.AgentOptions{},
}).Build(ctx)
if err != nil {
    log.Fatal(err)
}

myAgent, err := rt.NewAgent(ctx, cfg)
if err != nil {
    log.Fatal(err)
}

if err := myAgent.Init(ctx); err != nil {
    log.Fatal(err)
}

output, err := myAgent.Execute(ctx, &agent.Input{
    TraceID: "trace-001",
    Content: "Please analyze this code",
})
```

If you are wiring only the `agent` submodule runtime, use `agent/runtime.Builder` directly:

```go
import runtime "github.com/BaSui01/agentflow/agent/runtime"

opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EnableLSP = true

gateway := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

myAgent, err := runtime.NewBuilder(gateway, logger).
    WithOptions(opts).
    Build(ctx, cfg)
```

### 3. Tool Calling

The framework supports native tool calling with automatic ReAct loop handling LLM -> Tool -> LLM conversations.

```go
import "github.com/BaSui01/agentflow/llm/capabilities/tools"

registry := tools.NewDefaultRegistry(logger)
registry.Register("search", searchFunc, tools.ToolMetadata{
    Schema: llm.ToolSchema{
        Name:        "search",
        Description: "Search for information",
        Parameters:  json.RawMessage(`{"type":"object","properties":{"query":{"type":"string"}}}`),
    },
    Timeout: 30 * time.Second,
})

executor := tools.NewDefaultExecutor(registry, logger)
reactExecutor := tools.NewReActExecutor(provider, executor, tools.ReActConfig{
    MaxIterations: 10,
    StopOnError:   false,
}, logger)
```

## Data Flow Architecture

```text
User Input -> Agent.Execute()
    ->
Input Validation (Guardrails)
    ->
Build Messages (System Prompt + Context + User Input)
    ->
LLM Call (Provider.Completion/Stream)
    ->
[If Tool Calls] -> ReAct Loop
    ->                   ->
    ->              Tool Execution
    ->                   ->
    ->              Return Results to LLM
    ->                   ->
    <---------------------
    ->
Output Validation (Guardrails)
    ->
Save Memory
    ->
Return Result
```
