# Agent Development Tutorial

AgentFlow provides a complete agent development framework with state management, multi-layer memory, skills system, MCP integration, guardrails, advanced reasoning patterns, Human-in-the-Loop, async execution, voice interaction, and sandbox execution.

Entrypoint positioning first:

- Repository-level official entry: `sdk.New(opts).Build(ctx)`
- Official runtime entry for the `agent` submodule: `agent/runtime.Builder`
- `agent.NewAgentBuilder`, `agent.CreateAgent`, and `agent.BuildBaseAgent` are retained only as advanced extension surfaces

## Creating Agents

### Using `agent/runtime.Builder`

```go
import (
    runtime "github.com/BaSui01/agentflow/agent/runtime"
    "github.com/BaSui01/agentflow/types"
    "github.com/BaSui01/agentflow/llm/providers/openai"
)

provider := openai.NewOpenAIProvider(cfg, logger)
gateway := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EnableReflection = true
opts.EnableToolSelection = true

myAgent, err := runtime.NewBuilder(gateway, logger).WithOptions(opts).Build(ctx, types.AgentConfig{
    Core: types.CoreConfig{
        ID:   "assistant-001",
        Name: "Smart Assistant",
        Type: "assistant",
    },
    Model: types.ModelOptions{
        Model:       "gpt-4o",
        MaxTokens:   4096,
        Temperature: 0.7,
    },
    Control: types.AgentControlOptions{
        SystemPrompt: "You are a professional technical assistant",
        Reflection:    &types.ReflectionConfig{},
        ToolSelection: &types.ToolSelectionConfig{},
    },
    Tools: types.ToolProtocolOptions{
        AllowedTools: []string{"search", "calculator", "code_executor"},
        Handoffs:     []string{"review-agent", "research-agent"},
    },
})
```

If you want one repository-level runtime that assembles Agent, Workflow, and RAG together, prefer `sdk.New(opts).Build(ctx)`. This tutorial focuses on the `agent` submodule, so the examples default to `agent/runtime.Builder`.

Repository-level official entry example:

```go
rt, err := sdk.New(sdk.Options{
    Logger: logger,
    Agent:  &sdk.AgentOptions{},
}).Build(ctx)
```

Runtime configuration main surface:

- `types.AgentConfig` uses a three-layer runtime surface: `Model / Control / Tools`
- Runtime first normalizes `AgentConfig` into `ExecutionOptions`
- Provider-side `ChatRequest` objects are emitted through `ChatRequestAdapter`
- `ChatRequest` is a gateway/provider adapter DTO, not the formal Agent runtime config surface

### Agent Lifecycle

```go
// Create lifecycle manager
lifecycle := agent.NewLifecycleManager(myAgent, logger)

// Start (includes initialization and health checks)
if err := lifecycle.Start(ctx); err != nil {
    log.Fatal(err)
}

// Check running status
if lifecycle.IsRunning() {
    health := lifecycle.GetHealthStatus()
    fmt.Printf("Healthy: %v, State: %s\n", health.Healthy, health.State)
}

// Execute task
response, err := myAgent.Execute(ctx, &agent.Input{
    TraceID: "trace-001",
    Content: "Analyze this code",
    Variables: map[string]string{"language": "Go"},
})

// Restart
lifecycle.Restart(ctx)

// Stop
lifecycle.Stop(ctx)
```

```

         ↑         ↓
         └─────────┘
```

## Advanced Reasoning Patterns

The default runtime stays on the official `react` path, with `reflection` available as a quality enhancement. All other reasoning strategies now require explicit opt-in:

- `advanced`: `Reflexion`, `ReWOO`, `Plan-and-Execute`
- `experimental`: `Tree of Thought`, `Dynamic Planner`, `Iterative Deepening`

These strategies now rely on native structured output and/or native tool calling at runtime. They should not be documented or implemented as “prompt the model to return JSON text and parse it manually”.

Example:

```go
opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EnableReflection = true
opts.ReasoningExposure = agent.ReasoningExposureAdvanced // or agent.ReasoningExposureAll
```

## Human-in-the-Loop

### Approval Interrupt Manager

```go
import "github.com/BaSui01/agentflow/agent/hitl"

store := hitl.NewInMemoryInterruptStore()
manager := hitl.NewInterruptManager(store, logger)

go func() {
    for {
        interrupts := manager.GetPendingInterrupts("")
        for _, in := range interrupts {
            _ = manager.ResolveInterrupt(ctx, in.ID, &hitl.Response{
                OptionID: "approve",
                Approved: true,
                Comment:  "auto-approve demo",
            })
        }
        time.Sleep(100 * time.Millisecond)
    }
}()

response, err := manager.CreateInterrupt(ctx, hitl.InterruptOptions{
    WorkflowID:  "workflow-001",
    NodeID:      "step-3",
    Type:        hitl.InterruptTypeApproval,
    Title:       "Approval Required",
    Description: "Agent requests a sensitive operation",
    Options: []hitl.Option{
        {ID: "approve", Label: "Approve", IsDefault: true},
        {ID: "reject", Label: "Reject"},
    },
    Timeout: 5 * time.Minute,
})
if response != nil && response.Approved {
    // Execute operation
}
```

## Advanced Multi-Agent Modes

New code should continue to use `agent/team`. Common modes are:

| Mode | Use case |
|------|----------|
| `team.ModeSupervisor` | A supervisor decomposes work and gathers worker results. |
| `team.ModeRoundRobin` | Multiple agents iterate sequentially. |
| `team.ModeSelector` | A selector dynamically chooses the next agent. |
| `team.ModeSwarm` | Agents collaborate autonomously through handoff instructions. |

```go
t, err := team.NewTeamBuilder("research-team").
    AddMember(supervisorAgent, "supervisor").
    AddMember(researcherAgent, "researcher").
    AddMember(writerAgent, "writer").
    WithMode(team.ModeSupervisor).
    WithMaxRounds(4).
    Build(logger)

result, err := t.Execute(ctx, "Research Go concurrency and write a technical report")
```

Use `workflow/runtime` when you need explicit DAGs, branches, loops, or HITL gates. Do not import internal engine packages from tutorials or application code.

## Long-Running Executor

Days-level task execution with checkpoints and recovery.

```go
import "github.com/BaSui01/agentflow/agent/longrunning"

executor := longrunning.NewExecutor(longrunning.ExecutorConfig{
    CheckpointInterval: 5 * time.Minute,
    CheckpointDir:      "./checkpoints",
    MaxRetries:         3,
    AutoResume:         true,
}, logger)

steps := []longrunning.StepFunc{
    func(ctx context.Context, state any) (any, error) { return collectData(ctx, state) },
    func(ctx context.Context, state any) (any, error) { return processData(ctx, state) },
    func(ctx context.Context, state any) (any, error) { return generateReport(ctx, state) },
}

exec := executor.CreateExecution("Data Analysis", steps)
executor.Start(ctx, exec.ID, initialState)

// Check progress
exec, _ := executor.GetExecution(exec.ID)
fmt.Printf("Progress: %.2f%%, Step: %d/%d\n", exec.Progress, exec.CurrentStep, exec.TotalSteps)

// Pause/Resume
executor.Pause(exec.ID)
executor.Resume(exec.ID)

// Load from checkpoint
exec, _ = executor.LoadExecution(exec.ID)
```

## Agent Registry

Centralized agent type and factory management.

```go
import agent "github.com/BaSui01/agentflow/agent/runtime"

agent.InitGlobalRegistry(logger)

// Register custom agent type via the global registry
agent.GlobalRegistry.Register("custom_analyzer", func(
    config types.AgentConfig,
    gateway llmcore.Gateway,
    memory agent.MemoryManager,
    toolManager agent.ToolManager,
    bus agent.EventBus,
    logger *zap.Logger,
) (agent.Agent, error) {
    opts := runtime.DefaultBuildOptions()
    opts.EnableAll = false
    opts.MemoryManager = memory
    opts.ToolManager = toolManager
    opts.EventBus = bus

    return runtime.NewBuilder(gateway, logger).
        WithOptions(opts).
        Build(context.Background(), config)
})

// Create agent through the global registry extension path
myAgent, _ := agent.GlobalRegistry.Create(config, provider, memory, toolManager, bus, logger)

// Check if type is registered
if agent.GlobalRegistry.IsRegistered("custom_analyzer") {
    fmt.Println("Type registered")
}

// List all registered types
types := agent.GlobalRegistry.ListTypes()
```

Notes:

- The global registry create path is positioned as a registry typed-factory extension path.
- Normal integration should still start from `sdk.New(opts).Build(ctx)` or `agent/runtime.Builder`.
- If you already have a registry instance, call `AgentRegistry.Create(...)` directly instead of treating the package-level global-registry helper as a general-purpose constructor.

## Best Practices

1. **Use Builder Pattern**: Clear configuration of agent features
2. **Enable Guardrails**: Required for production environments
3. **Configure Memory Appropriately**: Choose memory types based on use case
4. **Modularize Skills**: Split complex capabilities into independent skills
5. **Monitor Events**: Subscribe to key events for debugging
6. **Enable Checkpoints**: Protect long-running tasks from data loss
7. **Keep `react` as the default**:
   - Default production path: `react` (+ optional `reflection`)
   - Advanced opt-in: `ReWOO`, `Reflexion`, `Plan-and-Execute`
   - Experimental only: `Tree of Thought`, `Dynamic Planner`, `Iterative Deepening`
8. **Human-in-the-Loop**: Enable approval for sensitive operations
9. **Async Execution**: Use for long tasks to avoid blocking
10. **Sandbox Execution**: Always run AI-generated code in sandbox
11. **Defensive Prompts**: Enable injection protection and output validation
12. **Use `agent/team` for multi-agent work**: Treat hierarchical/collaboration/crews as legacy or advanced compatibility surfaces
13. **Multi-Agent Collaboration**: Choose appropriate mode (Debate/Consensus/Pipeline)
14. **Crews Teams**: Define clear role responsibilities
15. **Smart Memory Management**: Enable intelligent decay to prevent memory bloat
16. **Federation**: Use federation orchestrator for cross-organization scenarios
17. **Containerized Deployment**: Use K8s or cloud services for production
