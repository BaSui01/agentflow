# Workflow Orchestration

AgentFlow now standardizes on a **single DAG workflow entry**.  
Legacy parallel entry types (`Chain / Parallel / Routing`) are retired from the public execution path.

The boundary between the official entry and advanced building blocks is now:

- Official execution entry: `workflow/runtime.Builder` -> `Facade.ExecuteDAG(...)`
- Official DSL entry: `workflow/runtime.Builder` -> `Parser`
- Advanced building blocks: `workflow.NewDAGExecutor(...)`, `(*workflow.DAGWorkflow).Execute(...)`

Those advanced building blocks still exist for framework internals and advanced extensions, but they are no longer the primary entrypoints used in this tutorial.

## 1. Minimal runnable example

```go
import (
    "github.com/BaSui01/agentflow/workflow"
    workflowruntime "github.com/BaSui01/agentflow/workflow/runtime"
)

extractStep := workflow.NewFuncStep("extract", func(ctx context.Context, input any) (any, error) {
    text := input.(string)
    return strings.ToUpper(text), nil
})

summarizeStep := workflow.NewFuncStep("summarize", func(ctx context.Context, input any) (any, error) {
    text := input.(string)
    return "SUMMARY: " + text, nil
})

wf, err := workflow.NewDAGBuilder("text-pipeline").
    WithDescription("extract -> summarize").
    AddNode("extract", workflow.NodeTypeAction).
        WithStep(extractStep).
        Done().
    AddNode("summarize", workflow.NodeTypeAction).
        WithStep(summarizeStep).
        Done().
    AddEdge("extract", "summarize").
    SetEntry("extract").
    Build()
if err != nil {
    return err
}

wfRuntime := workflowruntime.NewBuilder(nil, logger).
    WithDSLParser(false).
    Build()

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, "hello workflow")
```

## 2. Conditional branching

```go
wf, err := workflow.NewDAGBuilder("conditional").
    AddNode("check", workflow.NodeTypeCondition).
        WithCondition(func(ctx context.Context, input any) (bool, error) {
            score := input.(int)
            return score >= 60, nil
        }).
        WithOnTrue("pass").
        WithOnFalse("reject").
        Done().
    AddNode("pass", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("pass", func(ctx context.Context, input any) (any, error) {
            return "PASS", nil
        })).
        Done().
    AddNode("reject", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("reject", func(ctx context.Context, input any) (any, error) {
            return "REJECT", nil
        })).
        Done().
    AddEdge("check", "pass").
    AddEdge("check", "reject").
    SetEntry("check").
    Build()
```

## 3. Loops and error handling

```go
wf, err := workflow.NewDAGBuilder("retry-loop").
    AddNode("loop", workflow.NodeTypeLoop).
        WithLoop(workflow.LoopConfig{
            Type:          workflow.LoopTypeFor,
            MaxIterations: 3,
        }).
        Done().
    AddNode("work", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("work", func(ctx context.Context, input any) (any, error) {
            return input, nil
        })).
        WithErrorConfig(workflow.ErrorConfig{
            Strategy:   workflow.ErrorStrategyRetry,
            MaxRetries: 2,
        }).
        Done().
    AddEdge("loop", "work").
    SetEntry("loop").
    Build()
```

## 4. Checkpoints

```go
wfRuntime := workflowruntime.NewBuilder(checkpointManager, logger).
    WithHistoryStore(historyStore).
    Build()

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, input)
```

Notes:
- `checkpointManager` is optional (`nil` is valid).
- `workflow.NewDAGExecutor(...)` is now treated as an internal runtime building block; the official public path remains `workflow/runtime.Builder`.
- For long-running flows, pair checkpoints with execution history for recovery and auditing.

## 5. DSL / JSON / YAML integration

```go
wfRuntime := workflowruntime.NewBuilder(nil, logger).Build()

// YAML bytes -> DAGWorkflow
wf, err := wfRuntime.Parser.Parse([]byte(yamlText))

// File loading
wf, err = wfRuntime.Parser.ParseFile("workflow.yaml")

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, input)
```

Notes:

- `workflow/runtime.Builder` wires `Parser + Facade` onto the same runtime path, so DSL definition and DAG execution stay on one official chain.
- `workflow.FromYAML(...)`, `workflow.FromJSON(...)`, and `def.ToDAGWorkflow()` remain useful low-level conversion helpers, but they are no longer the primary tutorial path.

## 6. Node types

| Type | Purpose |
|---|---|
| `NodeTypeAction` | Execute a step |
| `NodeTypeCondition` | Conditional branch |
| `NodeTypeLoop` | Iteration |
| `NodeTypeParallel` | Concurrent branches within DAG |
| `NodeTypeSubGraph` | Nested subgraph |
| `NodeTypeCheckpoint` | Checkpoint node |

## 7. Recommended practices

1. Use DAG as the single execution entry.  
2. Keep each node focused on one responsibility.  
3. Configure explicit error strategy (`retry/skip/fail_fast`) for critical nodes.  
4. Use `workflow/runtime.Builder` to wire checkpoints and execution history instead of manually creating executors in business code.
