# Multimodal Framework API

This tutorial describes the framework-level multimodal endpoints exposed by AgentFlow.

These APIs are capability-oriented (not business-specific) and are designed to be composed by upper-layer products.

## Endpoints

- `GET /api/v1/multimodal/capabilities`
- `POST /api/v1/multimodal/references`
- `POST /api/v1/multimodal/image`
- `POST /api/v1/multimodal/video`
- `POST /api/v1/multimodal/plan`
- `POST /api/v1/multimodal/chat`

## Core Flow

1. Upload reference image to get `reference_id`.
2. Call image/video generation with `reference_id` to trigger image-guided generation.
3. Use `plan` for structured shot planning.
4. Use `chat` for single mode or `agent_mode`.

## Example

```bash
curl -X POST http://localhost:8080/api/v1/multimodal/references \
  -H "X-API-Key: your_key" \
  -F "file=@./reference.png"
```

```bash
curl -X POST http://localhost:8080/api/v1/multimodal/video \
  -H "X-API-Key: your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "A cinematic street scene with smooth movement",
    "provider": "runway",
    "advanced": true,
    "style_tokens": ["cinematic lighting", "high detail"],
    "reference_id": "ref_1740560000000000000"
  }'
```

## Configuration

Use `config.multimodal` for framework settings:

- `enabled`
- `reference_max_size_bytes`
- `reference_ttl`
- `reference_store_backend` (`redis` only)
- `reference_store_key_prefix`
- `default_image_provider`
- `default_video_provider`
- `image.*` and `video.*` provider keys

## Storage Strategy

Reference storage backend is `redis` and reuses `config.redis` connection settings.

For security, non-loopback Redis addresses should use `rediss://...` (TLS).
