# 目录导航（精简版）

用于减少"子目录过多"的阅读负担，先看这一层，再按需深入。

## 核心开发目录

- `sdk`：SDK 正式入口（`sdk.New(opts).Build(ctx)`）
- `cmd/agentflow`：程序入口与启动装配
- `agent`：Agent 核心能力（单 Agent runtime、多 Agent team）
- `llm`：模型、Provider、工具调用、路由
- `workflow`：工作流与 DAG
- `rag`：检索增强能力
- `api`：HTTP 协议层与处理器
- `config`：配置加载、校验、热重载
- `types`：零依赖核心类型定义（Layer 0）
- `pkg`：基础设施层（database/cache/metrics/middleware/telemetry 等），含 `pkg/migration` 数据库迁移
- `internal`：内部层
  - `internal/app/bootstrap`：启动装配 Builder 集合
  - `internal/usecase`：用例服务层（agent/chat/authorization/workflow/rag/tool）

## 测试与示例目录

- `testutil`：测试夹具与 mock
- `e2e`：端到端测试
- `benchmarks`：性能基准测试
- `examples`：示例工程

## 运维与工具目录

- `scripts`：脚本
- `docs`：文档

## 建议阅读顺序

1. 先看 `sdk` + `cmd/agentflow`（程序如何启动、SDK 如何装配）
2. 再看 `agent` + `llm`（核心行为）
3. 按需看 `workflow` / `rag`（场景能力）
4. 设计 Agent 框架或权限系统时，先看 `docs/architecture/README.md`，再进入对应专题文档
5. 最后看 `pkg` / `internal`（基础设施与装配细节）
