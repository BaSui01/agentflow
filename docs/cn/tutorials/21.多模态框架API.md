# 多模态框架 API

本文说明 AgentFlow 提供的**框架层**多模态接口。

这些接口是能力编排层，不绑定具体业务页面或行业模板，供上层应用自由组合。

## 接口列表

- `GET /api/v1/multimodal/capabilities`
- `POST /api/v1/multimodal/references`
- `POST /api/v1/multimodal/image`
- `POST /api/v1/multimodal/video`
- `POST /api/v1/multimodal/plan`
- `POST /api/v1/multimodal/chat`

## 典型流程

1. 上传参考图，获取 `reference_id`
2. 调用 `image` / `video` 时传入 `reference_id`，自动进入图生图 / 图生视频
3. 用 `plan` 生成结构化镜头计划
4. 用 `chat` 执行单阶段或 `agent_mode` 双阶段编排

## 调用示例

```bash
curl -X POST http://localhost:8080/api/v1/multimodal/references \
  -H "X-API-Key: your_key" \
  -F "file=@./reference.png"
```

```bash
curl -X POST http://localhost:8080/api/v1/multimodal/image \
  -H "X-API-Key: your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "A clean product shot with soft shadows",
    "provider": "openai",
    "advanced": true,
    "style_tokens": ["studio lighting"],
    "quality_tokens": ["high detail"],
    "reference_id": "ref_1740560000000000000"
  }'
```

## 配置说明

在 `config.multimodal` 下配置：

- `enabled`
- `reference_max_size_bytes`
- `reference_ttl`
- `reference_store_backend`（仅 `redis`）
- `reference_store_key_prefix`
- `default_image_provider`
- `default_video_provider`
- `image.*`、`video.*` Provider 密钥

## 存储建议

参考图存储后端为 `redis`，复用 `config.redis` 连接参数。

安全建议：非本机 Redis 地址请使用 `rediss://...`（TLS）。
