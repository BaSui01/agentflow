# 工作流编排

AgentFlow 当前推荐使用 **DAG 工作流单入口**。  
`Chain / Parallel / Routing` 旧并行入口已下线，请统一使用 `workflow.Facade.ExecuteDAG(...)`。

正式入口与高级构件的边界如下：

- 正式执行入口：`workflow/runtime.Builder` 产出的 `Facade.ExecuteDAG(...)`
- 正式 DSL 入口：`workflow/runtime.Builder` 产出的 `Parser`
- 高级构件：`workflow.NewDAGExecutor(...)`、`(*workflow.DAGWorkflow).Execute(...)`

上面的高级构件仍可用于框架内部装配或高级扩展，但不再作为本教程的正式主入口。

## 1. 最小可运行示例

```go
import (
    "github.com/BaSui01/agentflow/workflow"
    workflowruntime "github.com/BaSui01/agentflow/workflow/runtime"
)

extractStep := workflow.NewFuncStep("extract", func(ctx context.Context, input any) (any, error) {
    text := input.(string)
    return strings.ToUpper(text), nil
})

summarizeStep := workflow.NewFuncStep("summarize", func(ctx context.Context, input any) (any, error) {
    text := input.(string)
    return "SUMMARY: " + text, nil
})

wf, err := workflow.NewDAGBuilder("text-pipeline").
    WithDescription("extract -> summarize").
    AddNode("extract", workflow.NodeTypeAction).
        WithStep(extractStep).
        Done().
    AddNode("summarize", workflow.NodeTypeAction).
        WithStep(summarizeStep).
        Done().
    AddEdge("extract", "summarize").
    SetEntry("extract").
    Build()
if err != nil {
    return err
}

wfRuntime := workflowruntime.NewBuilder(nil, logger).
    WithDSLParser(false).
    Build()

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, "hello workflow")
```

## 2. 条件分支

```go
wf, err := workflow.NewDAGBuilder("conditional").
    AddNode("check", workflow.NodeTypeCondition).
        WithCondition(func(ctx context.Context, input any) (bool, error) {
            score := input.(int)
            return score >= 60, nil
        }).
        WithOnTrue("pass").
        WithOnFalse("reject").
        Done().
    AddNode("pass", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("pass", func(ctx context.Context, input any) (any, error) {
            return "PASS", nil
        })).
        Done().
    AddNode("reject", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("reject", func(ctx context.Context, input any) (any, error) {
            return "REJECT", nil
        })).
        Done().
    AddEdge("check", "pass").
    AddEdge("check", "reject").
    SetEntry("check").
    Build()
```

## 3. 循环与容错

```go
wf, err := workflow.NewDAGBuilder("retry-loop").
    AddNode("loop", workflow.NodeTypeLoop).
        WithLoop(workflow.LoopConfig{
            Type:          workflow.LoopTypeFor,
            MaxIterations: 3,
        }).
        Done().
    AddNode("work", workflow.NodeTypeAction).
        WithStep(workflow.NewFuncStep("work", func(ctx context.Context, input any) (any, error) {
            return input, nil
        })).
        WithErrorConfig(workflow.ErrorConfig{
            Strategy:   workflow.ErrorStrategyRetry,
            MaxRetries: 2,
        }).
        Done().
    AddEdge("loop", "work").
    SetEntry("loop").
    Build()
```

## 4. 检查点

```go
historyStore := workflow.NewExecutionHistoryStore()

wfRuntime := workflowruntime.NewBuilder(checkpointManager, logger).
    WithHistoryStore(historyStore).
    Build()

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, input)
```

说明：
- `checkpointManager` 可选；不需要检查点时可传 `nil`。
- `workflow.NewDAGExecutor(...)` 是 runtime 内部执行部件；正式入口仍是 `workflow/runtime.Builder`。
- 使用检查点时，推荐同时配置执行历史（`WithHistoryStore(...)`）用于恢复与审计。

## 5. DSL / JSON / YAML 接入

```go
wfRuntime := workflowruntime.NewBuilder(nil, logger).Build()

// YAML bytes -> DAGWorkflow
wf, err := wfRuntime.Parser.Parse([]byte(yamlText))

// 文件加载
wf, err = wfRuntime.Parser.ParseFile("workflow.yaml")

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, input)
```

说明：

- `workflow/runtime.Builder` 会统一装配 `Parser + Facade`，DSL 定义与执行都走同一条 runtime 主链。
- `workflow.FromYAML(...)`、`workflow.FromJSON(...)`、`def.ToDAGWorkflow()` 这类符号更适合底层定义转换，不再作为主教程入口。

## 6. 节点类型

| 类型 | 说明 |
|---|---|
| `NodeTypeAction` | 执行步骤 |
| `NodeTypeCondition` | 条件分支 |
| `NodeTypeLoop` | 循环迭代 |
| `NodeTypeParallel` | DAG 内并发分支 |
| `NodeTypeSubGraph` | 子图 |
| `NodeTypeCheckpoint` | 检查点 |

## 7. 实践建议

1. 统一走 DAG 单入口，不再引入并行旧入口封装。  
2. 每个节点只做一件事，降低重试与回放复杂度。  
3. 关键节点显式配置错误策略（`retry/skip/fail_fast`）。  
4. 长流程通过 `workflow/runtime.Builder` 统一挂接检查点与执行历史，不直接在业务侧 new executor。

## 进阶步骤类型

### 编排步骤（orchestration）

支持在 DAG 中直接编排多 Agent 协作：

```yaml
steps:
  multi_agent_review:
    type: orchestration
    orchestration:
      mode: deliberation
      agent_ids: [agent1, agent2, agent3]
      max_rounds: 5
      timeout_ms: 60000
```

### 工具链步骤（chain）

支持声明式工具链编排：

```yaml
steps:
  data_pipeline:
    type: chain
    chain:
      steps:
        - tool: read_file
          args: {path: "/data/input.json"}
        - tool: web_search
          arg_mapping: {query: "$.content"}
          on_error: skip
```

详细说明见 [10.工作流编排进阶](10.工作流编排进阶.md)。
