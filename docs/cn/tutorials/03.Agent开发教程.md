# Agent 开发教程

AgentFlow 提供完整的 Agent 开发框架，支持状态管理、多层记忆、技能系统、MCP 集成、安全护栏、高级推理模式、Human-in-the-Loop、异步执行等企业级特性。

入口定位先说明：

- 仓库级正式入口：`sdk.New(opts).Build(ctx)`
- `agent` 子模块正式 runtime 入口：`agent/runtime.Builder`
- `agent.NewAgentBuilder`、`agent.CreateAgent`、`agent.BuildBaseAgent` 仅作为高级扩展入口保留，不再与上述入口同级推荐

## 基础 Agent 创建

### 使用 `agent/runtime.Builder`

```go
import (
    runtime "github.com/BaSui01/agentflow/agent/runtime"
    "github.com/BaSui01/agentflow/types"
    "github.com/BaSui01/agentflow/llm/providers/openai"
)

// 创建 Provider
provider := openai.NewOpenAIProvider(cfg, logger)
gateway := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EnableReflection = true
opts.EnableToolSelection = true

myAgent, err := runtime.NewBuilder(gateway, logger).WithOptions(opts).Build(ctx, types.AgentConfig{
    Core: types.CoreConfig{
        ID:   "assistant-001",
        Name: "智能助手",
        Type: "assistant",
    },
    Model: types.ModelOptions{
        Model:       "gpt-4o",
        MaxTokens:   4096,
        Temperature: 0.7,
    },
    Control: types.AgentControlOptions{
        SystemPrompt: "你是一个专业的技术助手",
        Reflection:    &types.ReflectionConfig{},
        ToolSelection: &types.ToolSelectionConfig{},
    },
    Tools: types.ToolProtocolOptions{
        AllowedTools: []string{"search", "calculator", "code_executor"},
        Handoffs:     []string{"review-agent", "research-agent"},
    },
})

if err != nil {
    log.Fatal(err)
}
```

如果你希望由仓库统一装配 Agent、Workflow、RAG，再对外暴露稳定入口，请优先使用 `sdk.New(opts).Build(ctx)`；本教程聚焦 `agent` 子模块，因此默认示例使用 `agent/runtime.Builder`。

仓库级正式入口示例：

```go
rt, err := sdk.New(sdk.Options{
    Logger: logger,
    Agent:  &sdk.AgentOptions{},
}).Build(ctx)
```

运行时配置主面约定：

- `types.AgentConfig` 的正式主面是三层：`Model / Control / Tools`
- 运行时会先把 `AgentConfig` 收口为 `ExecutionOptions`
- 底层 provider 请求统一通过 `ChatRequestAdapter` 生成 `ChatRequest`
- `ChatRequest` 是 gateway/provider adapter DTO，不是 Agent 运行时正式配置主面

### Agent 生命周期

```go
// 创建生命周期管理器
lifecycle := agent.NewLifecycleManager(myAgent, logger)

// 启动（包含初始化和健康检查）
if err := lifecycle.Start(ctx); err != nil {
    log.Fatal(err)
}

// 检查运行状态
if lifecycle.IsRunning() {
    // 获取健康状态
    health := lifecycle.GetHealthStatus()
    fmt.Printf("健康: %v, 状态: %s\n", health.Healthy, health.State)
}

// 执行任务
response, err := myAgent.Execute(ctx, &agent.Input{
    TraceID: "trace-001",
    Content: "帮我分析这段代码",
    Variables: map[string]string{
        "language": "Go",
    },
})

// 重启
lifecycle.Restart(ctx)

// 停止
lifecycle.Stop(ctx)
```

### 状态机

Agent 有严格的状态转换：

```
Init → Ready ⇄ Running → Ready | Paused | Completed | Failed
         ↑         ↓
         └─────────┘
```

```go
// 状态定义
const (
    StateInit      State = "init"
    StateReady     State = "ready"
    StateRunning   State = "running"
    StatePaused    State = "paused"
    StateCompleted State = "completed"
    StateFailed    State = "failed"
)

// 检查状态
state := myAgent.State()
if state == agent.StateReady {
    // 可以执行任务
}

// 状态转换（带校验）
if err := myAgent.Transition(ctx, agent.StateRunning); err != nil {
    log.Printf("状态转换失败: %v", err)
}
```

### 提示词增强器

```go
import agent "github.com/BaSui01/agentflow/agent/runtime"

// 创建提示词增强器
enhancer := agent.NewPromptEnhancer(agent.PromptEngineeringConfig{
    UseChainOfThought:   true,  // 启用思维链
```

## 多层记忆系统

AgentFlow 提供两套记忆系统：增强记忆系统和分层记忆系统。

### 增强记忆系统 (EnhancedMemorySystem)

支持五种记忆类型：

| 记忆类型 | 存储后端 | 用途 |
|----------|----------|------|
| 短期记忆 | Redis/内存 | 最近的交互，带 TTL |
| 工作记忆 | 内存 | 当前任务上下文 |
| 长期记忆 | 向量数据库 | 持久化重要信息，支持语义搜索 |
| 情节记忆 | 时序数据库 | 时间序列事件 |
| 语义记忆 | 知识图谱 | 结构化知识（实体、关系） |

```go
import "github.com/BaSui01/agentflow/agent/capabilities/memory"

// 创建增强记忆系统
memorySystem := memory.NewEnhancedMemorySystem(
    shortTermStore,  // Redis 或内存存储
    workingStore,    // 内存存储
    vectorStore,     // Qdrant/Pinecone
    episodicStore,   // InfluxDB
    knowledgeGraph,  // Neo4j
    memory.EnhancedMemoryConfig{
        ShortTermTTL:          24 * time.Hour,
        ShortTermMaxSize:      100,
        WorkingMemorySize:     20,
        LongTermEnabled:       true,
        VectorDimension:       1536,
        EpisodicEnabled:       true,
        EpisodicRetention:     30 * 24 * time.Hour,
        SemanticEnabled:       true,
        ConsolidationEnabled:  true,
        ConsolidationInterval: 1 * time.Hour,
    },
    logger,
)

// 保存短期记忆
memorySystem.SaveShortTerm(ctx, agentID, "用户喜欢简洁回答", metadata)

// 保存长期记忆（向量化）
memorySystem.SaveLongTerm(ctx, agentID, content, embedding, metadata)

// 语义搜索长期记忆
results, _ := memorySystem.SearchLongTerm(ctx, agentID, queryVector, topK)

// 记录情节
memorySystem.RecordEpisode(ctx, &types.EpisodicEvent{
    AgentID:   agentID,
    Type:      "task_completed",
    Content:   "完成代码审查任务",
    Timestamp: time.Now(),
})

// 添加知识（实体和关系）
memorySystem.AddKnowledge(ctx, &memory.Entity{
    ID:   "user_123",
    Type: "user",
    Name: "张三",
    Properties: map[string]interface{}{
        "preference": "简洁",
    },
})

// 启动记忆整合（后台自动将短期记忆转移到长期记忆）
memorySystem.StartConsolidation(ctx)
```

## MCP 集成

Model Context Protocol (MCP) 客户端集成：

```go
import mcpproto "github.com/BaSui01/agentflow/agent/execution/protocol/mcp"

// 创建 MCP 客户端
client := mcpproto.NewMCPClient(reader, writer, logger)

// 连接服务器
client.Connect(ctx, "mcp://localhost:8080")

// 列出可用工具
tools, _ := client.ListTools(ctx)

// 调用工具
result, _ := client.CallTool(ctx, "search", map[string]interface{}{
    "query": "Go 并发最佳实践",
})

// 列出资源
resources, _ := client.ListResources(ctx)

// 读取资源
resource, _ := client.ReadResource(ctx, "file:///path/to/file")

// 订阅资源更新
updates, _ := client.SubscribeResource(ctx, "file:///config.json")
go func() {
    for update := range updates {
        fmt.Printf("资源更新: %v\n", update)
    }
}()

// 获取提示词模板
prompt, _ := client.GetPrompt(ctx, "code_review", map[string]string{
    "language": "Go",
})

// 批量调用工具
results, _ := client.BatchCallTools(ctx, []mcp.ToolCall{
    {Name: "search", Arguments: json.RawMessage(`{"query": "test"}`)},
    {Name: "read_file", Arguments: json.RawMessage(`{"path": "main.go"}`)},
})
```

## 安全护栏

### 验证器

```go
import "github.com/BaSui01/agentflow/agent/capabilities/guardrails"

// 创建护栏配置（通常用于 guardrails.NewCoordinator 或 BaseAgent.SetGuardrails）
config := guardrails.DefaultConfig()
config.MaxInputLength = 10000
config.BlockedKeywords = []string{"password", "secret"}
config.PIIDetectionEnabled = true
config.InjectionDetection = true
config.OnInputFailure = guardrails.FailureActionReject
config.OnOutputFailure = guardrails.FailureActionWarn

// 创建验证器链
chain := guardrails.NewValidatorChain(&guardrails.ValidatorChainConfig{
    Mode: guardrails.ChainModeCollectAll,
})

// 添加内置验证器
chain.Add(guardrails.NewLengthValidator(&guardrails.LengthValidatorConfig{
    MaxLength: 10000,
    Action:    guardrails.LengthActionReject,
}))
chain.Add(guardrails.NewKeywordValidator(&guardrails.KeywordValidatorConfig{
    BlockedKeywords: []string{"password"},
    Action:          guardrails.KeywordActionReject,
}))
chain.Add(guardrails.NewPIIDetector(&guardrails.PIIDetectorConfig{
    Action: guardrails.PIIActionWarn,
    EnabledTypes: []guardrails.PIIType{
        guardrails.PIITypeEmail,
        guardrails.PIITypePhone,
        guardrails.PIITypeIDCard,
        guardrails.PIITypeBankCard,
    },
}))
chain.Add(guardrails.NewInjectionDetector(&guardrails.InjectionDetectorConfig{
    CustomPatterns:   []string{`ignore previous instructions`, `system prompt`, `<script>`},
    EnabledLanguages: []string{"en", "zh", "universal"},
}))

// 验证输入
result, err := chain.Validate(ctx, userInput)
if err != nil {
    log.Printf("验证执行失败: %v", err)
}
if !result.Valid {
    for _, err := range result.Errors {
        log.Printf("验证失败: [%s] %s", err.Code, err.Message)
    }
}
```

### 自定义验证器

```go
type CustomValidator struct {
    name     string
    priority int
}

func (v *CustomValidator) Validate(ctx context.Context, content string) (*guardrails.ValidationResult, error) {
    result := guardrails.NewValidationResult()
    
    // 自定义验证逻辑
    if strings.Contains(content, "危险内容") {
        result.AddError(guardrails.ValidationError{
            Code:     "CUSTOM_ERROR",
            Message:  "检测到危险内容",
            Severity: guardrails.SeverityHigh,
        })
    }
    
    return result, nil
}

func (v *CustomValidator) Name() string     { return v.name }
func (v *CustomValidator) Priority() int    { return v.priority }

// 注册自定义验证器
registry := guardrails.NewValidatorRegistry()
registry.Register(&CustomValidator{name: "custom", priority: 100})
```

## 高级推理模式

默认 runtime 只走 `react` 官方主链；`reflection` 仍可作为质量增强启用。其余推理模式按显式 opt-in 分级：

- `advanced`：`Reflexion`、`ReWOO`、`Plan-and-Execute`
- `experimental`：`Tree of Thought`、`Dynamic Planner`、`Iterative Deepening`

启用方式：

```go
opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EnableReflection = true
opts.ReasoningExposure = agent.ReasoningExposureAdvanced // 或 agent.ReasoningExposureAll
```

下面的模式都属于显式启用能力，不再默认注入 runtime。

### Tree of Thought (ToT, experimental)

探索多条推理路径，选择最优解：

```go
import (
    "github.com/BaSui01/agentflow/agent/capabilities/reasoning"
    llmgateway "github.com/BaSui01/agentflow/llm/gateway"
)

gw := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

tot := reasoning.NewTreeOfThought(gw, toolExecutor, reasoning.TreeOfThoughtConfig{
    BranchingFactor: 3,      // 每步生成 3 个思路
    MaxDepth:        5,      // 最大深度 5 层
    BeamWidth:       2,      // 保留最优的 2 条路径
    EvaluationMode:  "self", // LLM 自评估
    PruneThreshold:  0.3,    // 低于 0.3 分的路径被剪枝
    Timeout:         120 * time.Second,
    ParallelEval:    true,   // 并行评估
}, logger)

result, _ := tot.Execute(ctx, "设计一个高并发的消息队列系统")

fmt.Printf("最终答案: %s\n", result.FinalAnswer)
fmt.Printf("置信度: %.2f\n", result.Confidence)
fmt.Printf("推理步骤: %d\n", len(result.Steps))
fmt.Printf("总 Token: %d\n", result.TotalTokens)
```

### ReWOO (advanced)

先规划后执行，适合工具密集型任务：

> 说明：`ReWOO`、`Plan-and-Execute`、`ToT`、`Dynamic Planner`、`Iterative Deepening` 现在依赖原生 structured output 和/或原生 tool calling 运行时，不再建议通过“提示词要求返回 JSON 文本后再手工解析”的方式实现。

```go

    MaxPlanSteps:    10,              // 最大计划步骤
    Timeout:         120 * time.Second,
    ParallelWorkers: 5,               // 并行执行工作线程

// 三阶段执行：Planner → Worker → Solver

// 结果包含计划步骤和观察结果
```


## 事件系统

```go
// 创建事件总线
bus := agent.NewEventBus()

// 事件类型
const (
    EventStateChange       EventType = "state_change"
    EventToolCall          EventType = "tool_call"
    EventFeedback          EventType = "feedback"
    EventAgentRunStart     EventType = "agent_run_start"
    EventAgentRunComplete  EventType = "agent_run_complete"
    EventAgentRunError     EventType = "agent_run_error"

    e := event.(*agent.StateChangeEvent)
    log.Printf("Agent %s: %s → %s", e.AgentID_, e.FromState, e.ToState)
})

// 订阅工具调用事件
bus.Subscribe(agent.EventToolCall, func(event agent.Event) {
    e := event.(*agent.ToolCallEvent)
    log.Printf("工具调用: %s, 阶段: %s", e.ToolName, e.Stage)
})

// 订阅运行完成事件
bus.Subscribe(agent.EventAgentRunComplete, func(event agent.Event) {
    e := event.(*agent.AgentRunCompleteEvent)
    log.Printf("运行完成: run=%s, total_tokens=%d", e.RunID, e.TotalTokens)
})

// 取消订阅
subscriptionID := bus.Subscribe(agent.EventFeedback, handler)
bus.Unsubscribe(subscriptionID)

// 使用事件总线的常规接入仍建议通过 runtime.Builder
opts := runtime.DefaultBuildOptions()
opts.EnableAll = false
opts.EventBus = bus

gateway := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

runtime.NewBuilder(gateway, logger).
    WithOptions(opts).
    Build(ctx, cfg)
```

## Agent 注册表

集中管理 Agent 类型和工厂。

```go
import agent "github.com/BaSui01/agentflow/agent/runtime"

// 初始化全局注册表
agent.InitGlobalRegistry(logger)

// 注册自定义 Agent 类型
agent.GlobalRegistry.Register("custom_analyzer", func(

    if err != nil {
        return nil, err
    }
    return baseAgent, nil
})

// 使用注册表创建 Agent（高级扩展入口）
myAgent, _ := agent.GlobalRegistry.Create(

        Core: types.CoreConfig{
            ID:   "analyzer-001",
```

说明：

- 全局 registry 的 create 路径定位为 typed factory 扩展入口，适合需要按类型分发构造逻辑的高级场景。

- 如果你已经拿到了 registry 实例，优先直接调用 `AgentRegistry.Create(...)`；不要再把包级的全局 registry 便捷包装当成通用构造入口。

## 最佳实践

1. **使用 Builder 模式**：清晰配置 Agent 各项功能
2. **启用安全护栏**：生产环境必须启用输入验证和输出过滤
3. **合理配置记忆**：根据场景选择合适的记忆类型和容量
4. **技能模块化**：将复杂能力拆分为独立技能
5. **事件监控**：订阅关键事件进行监控和调试
6. **默认坚持 `react` 主链**：
   - 生产默认：`react`（可叠加 `reflection`）
   - 高级显式启用：`ReWOO`、`Reflexion`、`Plan-and-Execute`
   - 实验能力：`Tree of Thought`、`Dynamic Planner`、`Iterative Deepening`
7. **Human-in-the-Loop**：敏感操作启用人工审批
8. **异步执行**：长任务使用异步执行避免阻塞
9. **防御性提示**：启用提示注入防护和输出验证
10. **多 Agent 优先用 `agent/team`**：支持 Supervisor、RoundRobin、Selector、Swarm 模式
11. **层级架构**：复杂系统使用 Supervisor-Worker 模式
12. **多 Agent 协作**：根据任务选择合适的协作模式
13. **智能记忆管理**：启用智能衰减避免记忆膨胀
14. **联邦协作**：跨组织场景使用联邦编排器

## toolProvider 双模型架构

AgentFlow 支持双模型架构：使用便宜模型（如 GPT-4o-mini）优先处理工具调用链路，贵模型（如 Anthropic Claude Opus）做最终内容生成，大幅降低成本。

### 架构原理

```
用户请求 → BaseAgent
              ├── 有工具调用 → toolProvider（便宜模型）执行 ReAct 循环
              │                  └── 原生 tool calling；非原生 provider 自动降级 XML tool-calling
              └── 无工具调用 → provider（贵模型）直接生成内容
```

### 配置方式

推荐仍通过 `agent/runtime.Builder` 先完成主 runtime 装配，再设置专用 `toolProvider`：

```go
ag, _ := runtime.NewBuilder(claudeProvider, logger).Build(ctx, types.AgentConfig{
    Core: types.CoreConfig{ID: "my-agent"},
    LLM:  types.LLMConfig{Model: "claude-opus-4-20250115"},
    Runtime: types.RuntimeConfig{
        Tools: []string{"web_search", "calculator"},
    },
})

// 设置工具调用 Provider（便宜模型）
ag.SetToolProvider(gpt4oMiniProvider)
```

### 工作流程（completion.go）

`ChatCompletion` 方法内部逻辑：

1. 如果有可用工具且设置了 `toolProvider`，ReAct 循环优先使用 `toolProvider`
2. provider 支持原生工具调用时走 native tool calling；不支持时由 gateway 自动切到 XML tool-calling fallback
3. 如果没有设置 `toolProvider`，退化为使用主 `provider`
4. 无工具场景直接使用主 `provider` 生成内容

> 当前实现位置：`agent/runtime/request_runtime.go`

## MaxReActIterations 可配置

ReAct 循环的最大迭代次数可通过 `AgentConfig` 配置：

```go
gateway := llmgateway.New(llmgateway.Config{ChatProvider: provider, Logger: logger})

ag, _ := runtime.NewBuilder(gateway, logger).Build(ctx, types.AgentConfig{
    Core: types.CoreConfig{ID: "my-agent"},
    Runtime: types.RuntimeConfig{
        MaxReActIterations: 10,  // 默认值取决于框架配置
        Tools:              []string{"web_search"},
    },
})
```

> 当 ReAct 循环达到最大迭代次数时，会返回当前已有的最佳结果而非报错。

## 当前 runtime 文件归口

在 2026-04-22 的 Agent 模块收口后，`agent` root 包已删除。教程中涉及的执行逻辑现在统一归口到 `agent/runtime`：

| 文件 | 职责 | 核心方法 |
|------|------|----------|
| `agent/runtime/request_runtime.go` | 对话请求、流式输出、Plan/Execute 主执行路径 | `ChatCompletion()`, `StreamCompletion()`, `Plan()`, `Execute()` |
| `agent/runtime/base_agent.go` | BaseAgent 基础结构、生命周期与 `Observe()` | `Init()`, `Teardown()`, `Observe()` |
| `agent/runtime/builder.go` | 正式 runtime 构造入口 | `NewBuilder(...).Build(...)` |

- 常规接入优先走 `sdk.New(opts).Build(ctx)`
- 仅在 `agent` 子模块内装配运行时时，使用 `agent/runtime.Builder`
