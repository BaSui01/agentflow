# Hosted 工具与 MCP

AgentFlow 提供 Hosted 工具，包括文件操作、Shell 命令和 MCP Client。`write_file`、`edit_file`、`run_command` 等高风险工具在服务端启用后统一由 `AgentToolingRuntime` 注册，执行前会经过 `AuthorizationService` 授权。

## 文件操作工具

支持 `read_file`、`write_file`、`edit_file`、`list_directory` 四个工具。

### 安全配置

- **AllowedPaths**：白名单路径，仅允许访问指定目录及其子目录；空列表会拒绝文件操作
- **MaxFileSize**：单文件最大字节数，默认 10MB

### 使用示例

通过工具调用时使用对应工具名：

```yaml
steps:
  read_config:
    type: tool
    tool: read_file
    config:
      path: "/data/config.json"

  list_data:
    type: tool
    tool: list_directory
    config:
      path: "/data"
      recursive: false
```

## Shell 命令工具

`run_command` 用于执行 Shell 命令。
该工具属于 `shell_command` 高风险资源，默认需要审批或显式授权。

### 安全配置

- **BlockedCmds**：命令黑名单，默认包含 `rm -rf`、`format`、`mkfs` 等危险命令
- **AllowedCmds**：命令白名单，非空时仅允许以白名单前缀开头的命令
- **Timeout**：执行超时，默认 30s，最大 300s

### 使用示例

```yaml
steps:
  run_script:
    type: tool
    tool: run_command
    config:
      command: "go build -o app ./cmd"
      working_directory: "/project"
      timeout_seconds: 60
```

## MCP Client

连接外部 MCP Server，动态发现并调用其提供的工具。

### StdioTransport（子进程模式）

通过标准输入/输出与子进程通信：

```go
import (
    "github.com/BaSui01/agentflow/agent/hosted"
    "github.com/BaSui01/agentflow/agent/protocol/mcp"
)

transport, err := mcp.NewStdioTransport("npx", "-y", "@modelcontextprotocol/server-filesystem", "/data")
if err != nil {
    return err
}
client := mcp.NewDefaultMCPClient(transport, logger)
// Initialize + 注册工具
err = hosted.RegisterMCPTools(ctx, mcp.AsMCPClientLike(client), registry)
```

### SSETransport（HTTP 模式）

通过 SSE 连接远程 MCP Server：

```go
transport := mcp.NewSSETransport("https://mcp.example.com/sse", mcp.WithAuthToken("token"))
client := mcp.NewDefaultMCPClient(transport, logger)
err = hosted.RegisterMCPTools(ctx, mcp.AsMCPClientLike(client), registry)
```

### 动态工具发现：RegisterMCPTools

`RegisterMCPTools` 会调用 MCP Server 的 `tools/list`，将返回的工具逐一注册到 `ToolRegistry`，后续可在工作流中按工具名调用。

## 配置参考

`config.yaml` 中的 `hosted_tools` 部分：

```yaml
hosted_tools:
  file_ops:
    enabled: true
    allowed_paths:
      - /data
      - /tmp/agentflow
    max_file_size: 10485760  # 10MB

  shell:
    enabled: true
    timeout: 30s
    blocked_cmds:
      - "rm -rf"
      - "format "
      - "mkfs"

  mcp:
    enabled: true
    command: "npx"
    args:
      - "-y"
      - "@modelcontextprotocol/server-filesystem"
      - "/data"
    # 或使用 base_url 连接 HTTP SSE 模式
    # base_url: "https://mcp.example.com/sse"

  approval:
    backend: "file"   # memory | file | redis
    grant_ttl: 15m
    scope: "request"   # request | agent_tool | tool
    persist_path: "./checkpoints/tool_approval_grants.json"
    redis_prefix: "agentflow:tool_approval"
```

- `approval.backend=memory`：仅进程内缓存，重启后失效。
- `approval.backend=file`：使用 `persist_path` 持久化到本地文件，适合单实例部署。
- `approval.backend=redis`：使用共享 Redis 键空间，适合多实例部署；复用 `config.redis` 连接参数。
- `approval.grant_ttl`：审批通过后的临时授权窗口，过期后需要重新审批。
- `approval.scope=request`：按 `agent_id + tool + args hash` 授权。
- `approval.scope=agent_tool`：按 `agent_id + tool` 授权，同一 agent 同工具不同参数可复用。
- `approval.scope=tool`：按工具名授权，作用域最宽，建议仅在可信环境使用。
- `approval.persist_path`：审批授权窗口的持久化文件。设置后，已批准的临时授权可跨进程重启恢复，直到 TTL 到期。
- `approval.redis_prefix`：Redis 后端键前缀。不同环境建议使用独立前缀隔离授权窗口。
- 运行中可通过 `/api/v1/tools/approvals/stats` 查看当前审批/授权窗口统计，通过 `/api/v1/tools/approvals/cleanup` 主动清理过期授权窗口。
- 运行中可通过 `/api/v1/tools/approvals/grants` 查看当前 active grants，并通过 `/api/v1/tools/approvals/grants/{fingerprint}` 手动撤销授权窗口。
- 运行中可通过 `/api/v1/tools/approvals/history` 查看审批请求、审批结果、授权撤销与过期清理等历史事件。
