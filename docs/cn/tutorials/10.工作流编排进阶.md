# 工作流编排进阶

本文介绍编排步骤、工具链步骤和 Checkpoint 持久化等进阶能力。

## 编排步骤（orchestration）

在 DAG 中直接编排多 Agent 协作，`type: orchestration`。

### mode 选项

| mode | 说明 |
|------|------|
| collaboration | 协作模式（辩论/共识/流水线/广播） |
| hierarchical | 分层模式（Supervisor-Worker） |
| crew | 角色组模式 |
| deliberation | 反思模式，多轮自我修正 |
| reasoning | 单 Agent 推理 |
| federation | 联邦模式 |
| loop | 循环迭代 |

### DSL YAML 示例

```yaml
steps:
  multi_agent_review:
    type: orchestration
    orchestration:
      mode: deliberation
      agent_ids: [agent1, agent2, agent3]
      max_rounds: 5
      timeout_ms: 60000
```

## 工具链步骤（chain）

声明式工具链编排，`type: chain`。步骤按顺序执行，上一步输出可作为下一步输入。

### arg_mapping

从上一步结果取值，使用 JSONPath 表达式（如 `$.content`、`$.result`）：

```yaml
chain:
  steps:
    - tool: read_file
      args: {path: "/data/input.json"}
    - tool: web_search
      arg_mapping: {query: "$.content"}
      on_error: skip
```

### on_error 策略

| 值 | 说明 |
|----|------|
| fail | 默认，出错即终止 |
| skip | 跳过当前步骤，继续执行 |
| retry | 按 max_retries 重试 |

### DSL YAML 示例

```yaml
steps:
  data_pipeline:
    type: chain
    chain:
      steps:
        - tool: read_file
          args: {path: "/data/input.json"}
        - tool: web_search
          arg_mapping: {query: "$.content"}
          on_error: skip
        - tool: write_file
          arg_mapping:
            path: "$.output_path"
            content: "$.snippet"
```

## Checkpoint 持久化

工作流执行状态可持久化到 PostgreSQL，支持恢复与审计。

### PostgreSQL 后端配置

```yaml
workflow_checkpoint:
  backend: postgres

database:
  driver: postgres
  host: localhost
  port: 5432
  user: agentflow
  password: ""
  name: agentflow
  ssl_mode: require
```

表 `workflow_checkpoints` 由 `PostgreSQLCheckpointStore` 自动创建，包含 `id`、`workflow_id`、`thread_id`、`version`、`data`（JSONB）、`created_at`、`updated_at`。
