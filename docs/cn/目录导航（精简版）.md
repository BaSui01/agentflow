# 目录导航（精简版）

用于减少“子目录过多”的阅读负担，先看这一层，再按需深入。

## 核心开发目录

- `cmd/agentflow`：程序入口与启动装配
- `agent`：Agent 核心能力
- `llm`：模型、Provider、工具调用、路由
- `workflow`：工作流与 DAG
- `rag`：检索增强能力
- `api`：HTTP 协议层与处理器
- `config`：配置加载、校验、热重载
- `types`：基础类型定义
- `pkg`：可复用基础设施（middleware/server/telemetry 等）
- `internal`：内部启动/装配代码（不对外暴露）

## 测试与示例目录

- `tests`：端到端/契约/集成测试
- `testutil`：测试夹具与 mock
- `examples`：示例工程

## 运维与工具目录

- `migrations`：数据库迁移
- `scripts`：脚本
- `tools`：代码生成或工具集成
- `docs`：文档

## 建议阅读顺序

1. 先看 `cmd/agentflow`（程序如何启动）
2. 再看 `agent` + `llm`（核心行为）
3. 按需看 `workflow` / `rag`（场景能力）
4. 最后看 `pkg` / `internal`（基础设施与装配细节）
