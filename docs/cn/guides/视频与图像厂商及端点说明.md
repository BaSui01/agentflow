# 视频与图像厂商及端点说明

> 更新时间：2026-04-21  
> 口径：本文聚焦 **项目当前已接入的图像 / 视频厂商**、默认 Base URL、主要 endpoint、流式 / 轮询方式，以及配置复用关系。  
> 如果你要看“最近 12 个月的官方主流图像 / 视频 / 语音模型”，请优先看 `docs/cn/guides/近12个月主流多模态模型总表.md`。  
> 如果你要看统一的 provider `/models`、chat / image / video / speech 总览，请看 `./模型与媒体端点参考.md`。

---

## 1. 接入清单（配置即用）

| 模态 | 已完整接入（配置 API Key 即可用） | 备注 |
|---|---|---|
| 图像 | `openai`、`gemini`、`flux`、`stability`、`ideogram`、`tongyi`、`zhipu`、`baidu`、`doubao`、`tencent`、`kling` | 通过 `/api/v1/multimodal/image` 使用 |
| 视频 | `sora`、`runway`、`veo`、`kling`、`luma`、`minimax-video`、`seedance`、Grok（经 chat/video 能力） | 通过 `/api/v1/multimodal/video` 使用 |

- **已接入**：在 `config` 中配置对应 `*_api_key`（及可选 `*_base_url`）后，`GET /api/v1/multimodal/capabilities` 会返回该 provider。
- **一个 Key 双用**：OpenAI（图像 + Sora 视频）、Google（Gemini 图像 + Veo 视频）、可灵 Kling（图像 + 视频）都支持“一个 key 同时启用两种模态”。
- **字节系拆分**：视频使用 `seedance`（即梦），图像使用 `doubao`（豆包 / SeeDream），两者是不同产品与 endpoint。

## 3. 共享 Key / 配置关系

| 组合 | 说明 | 当前配置方式 |
|---|---|---|
| OpenAI 图像 + Sora 视频 | 同一个 OpenAI API Key 可同时启用图像与视频 | `openai_api_key`；未单配 `sora_api_key` 时复用 |
| Gemini 图像 + Veo 视频 | 同一个 Google API Key 可同时启用 Gemini 图像与 Veo 视频 | `google_api_key` / `gemini_api_key` |
| Kling 图像 + 视频 | 同一个 Kling API Key 可同时做图像和视频 | `kling_api_key` |
| 字节图像 + 视频 | 豆包图像与即梦视频是两条产品线 | `doubao_api_key` + `seedance_api_key` 分开配置 |

## 4. 按模态的典型调用路径

### 图像
- 查询能力：`GET /api/v1/multimodal/capabilities`
- 生成图像：`POST /api/v1/multimodal/image`
- 流式图像：请求体带 `stream: true`，服务端返回 SSE
- 引用图上传：`POST /api/v1/multimodal/references`

### 视频
- 查询能力：`GET /api/v1/multimodal/capabilities`
- 生成视频：`POST /api/v1/multimodal/video`
- 可灵回调：请求体可带 `callback_url`
- 大多数视频模型为**异步任务**，需要轮询或回调

## 5. 配置字段（速查）

| 模态 | 常见配置键 | 说明 |
|---|---|---|
| 图像 | `OPENAI_API_KEY`、`GEMINI_API_KEY`、`FLUX_API_KEY`、`STABILITY_API_KEY`、`IDEOGRAM_API_KEY`、`TONGYI_API_KEY`、`ZHIPU_API_KEY`、`BAIDU_API_KEY`、`DOUBAO_API_KEY`、`TENCENT_SECRET_ID/KEY`、`KLING_API_KEY` | 图像厂商 API Key / 凭证 |
| 视频 | `SORA_API_KEY`、`RUNWAY_API_KEY`、`GOOGLE_API_KEY`、`KLING_API_KEY`、`LUMA_API_KEY`、`MINIMAX_API_KEY`、`SEEDANCE_API_KEY` | 视频厂商 API Key / 凭证 |
| 默认路由 | `DEFAULT_IMAGE_PROVIDER`、`DEFAULT_VIDEO_PROVIDER` | 默认图像 / 视频 provider |

> 完整字段仍以 `config/loader.go` 中 `MultimodalConfig`、`MultimodalImageConfig`、`MultimodalVideoConfig` 的 `env` 标签为准。
