# 模型厂商与模型中文命名规范

> 更新时间：2026-04-24  
> 适用范围：`docs/`、`README.md`、`AGENTS.md`、中文教程、评审说明、提交说明中涉及 LLM 厂商或模型命名的内容。  
> 目标：统一“供应商代码 / 品牌名 / 产品名 / 模型 ID / latest 表述”的写法，减少 `Anthropic` / `Claude`、`xAI` / `Grok`、`Google` / `Gemini` 等混写。

---

## 1. 总规则

1. **代码名不翻译**：目录名、配置键、环境变量、Provider code、API model id 一律保留原始英文，例如 `openai`、`anthropic`、`gemini`、`gpt-5.4`、`claude-opus-4-7`。
2. **中文正文用“品牌/产品 + 英文代码”双轨表达**：首次提及时优先写“中文推荐名 + 代码/模型 ID”，后文再用简称。
3. **模型 ID 必须放反引号**：例如 `gpt-5.4`、`gemini-3.1-pro-preview`、`qwen3-max`。
4. **不要翻译模型后缀**：`Preview`、`Beta`、`Flash`、`Pro`、`Opus`、`Sonnet`、`Haiku`、`Reasoning`、`Thinking`、日期版本后缀都保留英文原样。
5. **“最新模型”必须带日期**：凡写“当前最新 / 当前主推 / latest”，必须注明“截至 YYYY-MM-DD”并附官方来源；否则一律改写为“当前文档记录的模型”或“示例模型”。
6. **如果官方发布与 API 可用性不同步，必须同时写清可用性状态**：例如“已在 ChatGPT / Codex 发布，API availability coming soon”，不能把产品侧发布直接写成“当前 API 默认模型”。
7. **区分厂商与产品**：
   - `anthropic` 是厂商品牌，`Claude` 是产品线；首次提及写 **Anthropic Claude**。
   - `grok` 是产品线，厂商是 `xAI`；首次提及写 **xAI Grok**。
   - `gemini` 是产品线，品牌通常写 **Google Gemini**。
8. **项目内中文命名以本文件为准**：如果历史文档、截图、issue、旧计划与本规范冲突，以本文件为当前中文命名基线。

## 4. 推荐写法模板

### 4.1 首次提及厂商

- ✅ OpenAI（provider: `openai`）当前 API 默认推荐使用 `gpt-5.4`；`gpt-5.5` 已在 ChatGPT / Codex 发布，但 API availability coming soon。
- ✅ Anthropic Claude（provider: `anthropic`）当前模型线包括 `claude-opus-4-7` 与 `claude-sonnet-4-6`。
- ✅ Google Gemini（provider: `gemini`）在高端档可写 `gemini-3.1-pro-preview` 或 `gemini-2.5-pro`。
- ✅ xAI Grok（provider: `grok`）当前旗舰命名为 `grok-4.20`。
- ✅ 通义千问 Qwen（provider: `qwen`）当前可写 `qwen3-max`。

### 4.2 首次提及模型

- ✅ 默认示例模型使用 `gpt-5.4`（OpenAI GPT-5.4）。
- ✅ 如果要提 OpenAI 下一代产品侧发布状态，写成“`gpt-5.5`（截至 2026-04-24 已在 ChatGPT / Codex 发布，API availability coming soon）”。
- ✅ 如果需要引用官方 Preview 名称，请直接写 `gemini-3.1-pro-preview`。
- ✅ 带日期的版本号（如 `qwen3-max-2026-01-23`）必须原样保留。

### 4.3 不推荐写法

- ❌ `谷歌 3.1 Pro 预览版`
- ❌ `Anthropic 4.7`
- ❌ `xAI 4.20`
- ❌ `Qwen3 Max 0123`
- ❌ `OpenAI 最新模型`（没有日期和来源）

## 5. 写文档时的落地约束

1. **如果是代码/配置示例**：写 provider code 和 model id，不写中文翻译版模型名。
2. **如果是教程/说明文**：首提写中文推荐名 + 英文 ID；后续可只写品牌或产品简称。
3. **如果是能力矩阵/对比表**：厂商列统一使用本文件“推荐中文名”，模型列统一使用反引号中的官方 ID。
4. **如果是“最新模型”对比**：必须在表头或标题注明“截至 2026-04-24”这类绝对日期。
5. **如果官方同时给品牌名和产品名**：优先保留两者，例如 Anthropic Claude、Google Gemini、xAI Grok。
6. **如果厂商文档给了 capabilities / stage / deprecation 信息**：正文里不要只记模型名，最好同时说明“Preview / Stable / Deprecated / API coming soon”等状态。

---

## 6. 维护方式

- 上游厂商更新快；凡涉及“最新 / 当前主推 / 旗舰 / preview”表述，更新本文档时必须同步更新日期与官方来源。
- 如果官方页面同时包含“产品侧已发布”和“API 侧尚未开放”，以 **API 语境** 为准写项目文档，并在备注里补可用性状态。
- 如果只是在项目代码里引用模型 ID，不需要改本文档；只有“中文命名规则”变化或“官方主推型号”变化时才更新。
- 与本规范冲突的历史文档可以逐步修正，但新增文档必须直接遵守本规范。
