> 注意：截至 2026-04-24 的模型发布快照。官方最新信息请以厂商文档为准。  
> 时间范围：2025-04-21 ~ 2026-04-21

## 5. 公开 benchmark / 评测口径（web 核验）

> 本节基于 2026-04-24 对官方页面及其关联外链的二次 web 核验。只有满足“**来源可公开访问 + 指标名称明确 + 模型版本明确**”的内容，才适合写成 benchmark 事实；否则本文只保留模型家族与官方来源，不硬写分数。

|---|---|---|---|

### 5.2 当前不建议硬写具体分数的家族

1. **Google Gemini 3 / 2.5**：当前可抓取的官方 API docs / 模型页主要提供型号、能力、preview/stable 状态和限制信息；本轮可访问 HTML 中没有稳定 benchmark 总表。`Claude 4` 附录引用了 `Gemini 2.5 Pro Preview` model card，但当前直链抓取返回 `403`，因此本文暂不硬写具体分数。
2. **xAI Grok / Mistral / Magistral / Kimi / Qwen / GLM / 腾讯混元 / MiniMax / 豆包**：当前公开页主要是 models/pricing、模型列表、产品说明或云文档入口，不是稳定 benchmark 发布页。
3. **DeepSeek V4**：`DeepSeek V4 Preview Release` 页面明确给出 benchmark 图表，但当前文本抓取只能拿到相对表述，不能稳定提取逐项分数；在没有可复核文本或 tech report 对应页码前，不建议把图里的数字硬录入正文。

### 5.3 多模态 benchmark 使用建议

1. **对话 / 推理** 优先使用公开 benchmark，如 `SWE-Bench`、`Terminal-bench`、`BrowseComp`、`GPQA`、`Humanity's Last Exam`，并同时写明是否开 tools / high compute。
2. **STT** 优先使用 $WER$ 或官方明确声明的相对 $WER$ 改善；不要把“支持语言数”误写成精度指标。
3. **图像 / 视频 / TTS / Realtime** 当前官方页面更多是能力说明、限制、价格和示例；如果缺少稳定跨厂公开 benchmark，应明确写“**缺少稳定可比公开 benchmark**”，而不是硬补分数。
4. 如果后续确实要补第三方榜单（例如 `GDPval-AA` 一类），必须在表中显式标注“**第三方**”，并带上榜单日期与模型版本。
