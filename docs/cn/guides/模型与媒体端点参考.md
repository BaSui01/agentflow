# Provider Models / Media Endpoints 参考文档

> 更新时间：2026-04-24
> 注意：截至 2026-04-24 的模型快照。项目 fallback 仍按代码实际默认值，不一定立即跟随厂商最新发布。  
> 范围：结合 AgentFlow 当前 provider/多模态接入面，汇总最近 12 个月（2025-04-21 ~ 2026-04-21）主流模型家族、`/models` 端点与主流图像/视频/语音模型。  
> 说明：本文档区分 **项目当前 fallback / 路由基线** 与 **厂商官方近 12 个月主流模型**，两者不要混淆。

> 如果你关心“上游字段应该落到 `types.AgentConfig{Model, Control, Tools}` 的哪里”，请同时阅读 [`./模型字段与Agent框架接入指南.md`](./模型字段与Agent框架接入指南.md)。

> benchmark 口径补充：如果你关心**可公开复核的 benchmark / eval**，请优先看 `./近12个月主流多模态模型总表.md` 第 5 节。本文主要承担 **models / media endpoints / fallback / 路由基线** 索引职责，不把 endpoint 总览和 benchmark 排行混为一体。

## 6. 当前维护规则

1. **把 latest 当成带日期的事实，不当成永久常量。**
2. **把项目 fallback 和厂商官方 latest 分开写。**
3. **新增模型先判断归属：native provider 还是 compat provider。**
4. **compat 厂商优先在 `llm/providers/vendor/chat_profiles.go` 做 request hook / validation。**
5. **新增多模态主流模型时，至少同步：**
   - `docs/en/tutorials/02.ProviderConfiguration.md`
   - `docs/en/guides/RecentModelFamiliesAndModalities.md`
   - `./模型与媒体端点参考.md`
6. **如果字段语义发生变化或新增 provider-native 关键字段**（例如 `phase`、`thinkingLevel`、`adaptive thinking`、`responseJsonSchema`），同步更新 `./模型字段与Agent框架接入指南.md`，不要只改模型名不改接入说明。
7. **benchmark 事实优先收口到 `近12个月主流多模态模型总表.md` 第 5 节**；本文件只保留“端点 / Base URL / fallback / 路由注意点”的稳定索引。
8. **图像 / 视频 / TTS / Realtime 缺少稳定跨厂公开 benchmark 时，不要在这里补伪精确分数。**
9. **若确需补 benchmark，只写带绝对日期、模型版本、评测名和来源类型（官方 / 第三方）的数据。**
