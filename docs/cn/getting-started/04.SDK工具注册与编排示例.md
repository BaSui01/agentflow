# SDK 工具注册与编排示例

本文收口 SDK 级推荐路径。新项目默认从 `sdk.New(opts).Build(ctx)` 开始；单 Agent 由 `rt.NewAgent(...)` 创建，多 Agent 使用 `agent/team`，确定性 DAG 编排使用 `workflow/runtime`。

完整可运行示例见：

```bash
go run ./examples/22_sdk_official_surface
```

## 推荐路径

| 场景 | 推荐入口 |
|------|----------|
| 仓库级装配 | `sdk.New(opts).Build(ctx)` |
| 单 Agent | `rt.NewAgent(ctx, types.AgentConfig{...})` |
| 工具注册 / function calling | `sdk.AgentOptions.ToolManager` |
| 检索上下文 | `sdk.AgentOptions.RetrievalProvider` |
| 多 Agent 协作 | `agent/team.NewTeamBuilder(...)` |
| 显式 workflow | `workflow/runtime.NewBuilder(...).Build()` |

## SDK 注入工具与 ToolProvider

SDK 层的工具注册表面是 `sdk.AgentOptions.ToolManager`。`ToolManager` 只需要暴露工具 schema 和执行方法：

```go
type ToolManager interface {
    GetAllowedTools(agentID string) []types.ToolSchema
    ExecuteForAgent(ctx context.Context, agentID string, calls []types.ToolCall) []llmtools.ToolResult
}
```

最小装配示例：

```go
buildOpts := agentruntime.BuildOptions{
    MaxReActIterations: 3,
    MaxLoopIterations:  3,
}

rt, err := sdk.New(sdk.Options{
    Logger: logger,
    LLM: &sdk.LLMOptions{
        Provider:     mainProvider,
        ToolProvider: toolProvider,
    },
    Agent: &sdk.AgentOptions{
        BuildOptions: buildOpts,
        ToolManager:  toolManager,
        ToolScope:    []string{"get_weather"},
    },
}).Build(ctx)
```

`ToolProvider` 是可选项，用于把工具调用链路和主回答 provider 分开。工具闭环通常至少需要 2 轮模型调用，所以 function calling 示例应显式设置 `MaxReActIterations`。

## 注册 Function Calling 工具

```go
registry := llmtools.NewDefaultRegistry(logger)

_ = registry.Register("get_weather", getWeather, llmtools.ToolMetadata{
    Schema: types.ToolSchema{
        Type:        types.ToolTypeFunction,
        Name:        "get_weather",
        Description: "Get current weather for a city",
        Parameters:  json.RawMessage(`{"type":"object","properties":{"city":{"type":"string"}},"required":["city"]}`),
    },
})

toolManager := &myToolManager{
    registry: registry,
    executor: llmtools.NewDefaultExecutor(registry, logger),
}
```

Agent 创建和执行仍走 SDK runtime：

```go
ag, err := rt.NewAgent(ctx, types.AgentConfig{
    Core: types.CoreConfig{ID: "assistant", Name: "Assistant", Type: "assistant"},
    LLM:  types.LLMConfig{Model: "gpt-4o-mini"},
})
if err != nil {
    return err
}
if err := ag.Init(ctx); err != nil {
    return err
}

out, err := ag.Execute(ctx, &agentruntime.Input{
    Content: "Use a tool to check the weather.",
})
```

## 检索上下文与 Retrieval Tool

被动检索上下文使用 `RetrievalProvider`：

```go
type myRetrievalProvider struct{}

func (myRetrievalProvider) Retrieve(ctx context.Context, query string, topK int) ([]types.RetrievalRecord, error) {
    return []types.RetrievalRecord{{
        DocID:   "sdk-entrypoint",
        Content: "AgentFlow official entrypoint is sdk.New(opts).Build(ctx).",
        Source:  "kb",
        Score:   0.98,
    }}, nil
}
```

需要让模型显式决定何时检索时，把检索封装为普通 function tool，并仍通过同一个 `ToolManager` 注入：

```go
_ = registry.Register("search_knowledge", searchKnowledge, llmtools.ToolMetadata{
    Schema: types.ToolSchema{
        Type:        types.ToolTypeFunction,
        Name:        "search_knowledge",
        Description: "Search local knowledge",
        Parameters:  json.RawMessage(`{"type":"object","properties":{"query":{"type":"string"}},"required":["query"]}`),
    },
})

rt, err := sdk.New(sdk.Options{
    LLM: &sdk.LLMOptions{Provider: provider},
    Agent: &sdk.AgentOptions{
        BuildOptions:      buildOpts,
        ToolManager:       toolManager,
        RetrievalProvider: myRetrievalProvider{},
        ToolScope:         []string{"search_knowledge"},
    },
}).Build(ctx)
```

## Team：Supervisor / Selector

多 Agent 使用 `agent/team`，不要导入 Team 内部 engine 包。

```go
supervisorTeam, err := team.NewTeamBuilder("delivery").
    AddMember(supervisorAgent, "supervisor").
    AddMember(workerAgent, "worker").
    WithMode(team.ModeSupervisor).
    WithMaxRounds(3).
    Build(logger)

result, err := supervisorTeam.Execute(ctx, "分析需求并给出交付方案")
```

Selector 模式同样只切换 mode：

```go
selectorTeam, err := team.NewTeamBuilder("selector").
    AddMember(selectorAgent, "selector").
    AddMember(workerAgent, "worker").
    WithMode(team.ModeSelector).
    WithSelectorPrompt("Choose the next role by task fit.").
    Build(logger)
```

## Workflow：调用 Agent Step

确定性流程使用 `workflow/runtime`。在 DAG action node 中调用 Agent 时，Agent 仍来自 SDK runtime：

```go
agentStep := workflow.NewFuncStep("agent-step", func(ctx context.Context, input any) (any, error) {
    out, err := ag.Execute(ctx, &agentruntime.Input{Content: fmt.Sprint(input)})
    if err != nil {
        return nil, err
    }
    return out.Content, nil
})

graph := workflow.NewDAGGraph()
graph.AddNode(&workflow.DAGNode{ID: "ask-agent", Type: workflow.NodeTypeAction, Step: agentStep})
graph.SetEntry("ask-agent")

wf := workflow.NewDAGWorkflow("agent-workflow", "Call an Agent from a workflow node", graph)
wfRuntime := workflowruntime.NewBuilder(nil, logger).WithDSLParser(false).Build()

result, err := wfRuntime.Facade.ExecuteDAG(ctx, wf, "Summarize the SDK entrypoint.")
```

## 高级路径边界

推荐用户文档只展示上面的路径。

- `agent/runtime.Builder`：适合在 `agent` 子模块或框架内部做更细粒度 runtime 装配。
- `workflow/runtime.Builder`：是 workflow 官方装配入口，可在需要 DSL、history、checkpoint、circuit breaker 时直接使用。
- `internal/app/bootstrap`：服务端组合根，只负责 server 启动装配，不作为 SDK 用户教程入口。
- Team 内部 engine 包：实现细节，教程、examples 和业务代码都不应导入。
