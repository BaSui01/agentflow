# 📦 安装与配置

> 详细的安装步骤和配置选项说明

## 📋 系统要求

| 要求 | 最低版本 | 推荐版本 |
|-----|---------|---------|
| Go | 1.24.0 | 1.24+ |
| 操作系统 | Linux / macOS / Windows | - |
| 内存 | 512MB | 2GB+ |

## 🔧 安装方式

### 方式一：Go Modules（推荐）

```bash
# 在你的项目中
go get github.com/BaSui01/agentflow
```

### 方式二：从源码安装

```bash
git clone https://github.com/BaSui01/agentflow.git
cd agentflow
go build ./...
```

### 方式三：指定版本

```bash
# 安装特定版本
go get github.com/BaSui01/agentflow@v1.0.0

# 安装最新开发版
go get github.com/BaSui01/agentflow@latest
```

## 🔑 API Key 配置

AgentFlow 支持 13+ LLM 提供商，以下是各提供商的配置方式：

### 环境变量配置（推荐）

创建 `.env` 文件：

```bash
# ============ 国际提供商 ============
# OpenAI
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://api.openai.com

# Anthropic Claude
ANTHROPIC_API_KEY=sk-ant-xxx

# Google Gemini
GEMINI_API_KEY=xxx

# Mistral
MISTRAL_API_KEY=xxx

# xAI Grok
GROK_API_KEY=xxx

# ============ 国产提供商 ============
# DeepSeek
DEEPSEEK_API_KEY=sk-xxx

# 通义千问 Qwen
QWEN_API_KEY=sk-xxx

# 智谱 GLM
GLM_API_KEY=xxx

# 豆包（字节/火山引擎）
DOUBAO_API_KEY=xxx

# Kimi（月之暗面）
KIMI_API_KEY=xxx

# 腾讯混元
HUNYUAN_API_KEY=xxx

# MiniMax
MINIMAX_API_KEY=xxx
MINIMAX_GROUP_ID=xxx
```

### 代码中配置

```go
import (
    "github.com/BaSui01/agentflow/llm/providers"
    "github.com/BaSui01/agentflow/llm/providers/openai"
)

// 方式 1: 直接配置
provider := openai.NewOpenAIProvider(providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey:  os.Getenv("OPENAI_API_KEY"),
        BaseURL: "https://api.openai.com",
        Model:   "gpt-4o",
    },
}, logger)

// 方式 2: 从环境变量读取
provider := openai.NewOpenAIProvider(providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey:  os.Getenv("OPENAI_API_KEY"),
        BaseURL: os.Getenv("OPENAI_BASE_URL"),
        Model:   "gpt-4o",
    },
}, logger)
```

## ⚙️ 高级配置

```go
import "time"

config := providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
```

```go
config := providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey: os.Getenv("OPENAI_API_KEY"),
    },
}
```

### 自定义 Base URL（兼容 API）

```go
// 使用 OpenAI 兼容的第三方服务
config := providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey:  os.Getenv("API_KEY"),
        BaseURL: "https://your-proxy.com/v1",  // 自定义端点
        Model:   "gpt-4o",
    },
}
```

### 双模型架构（工具调用优化）

```go
// 主模型用于内容生成
mainProvider := openai.NewOpenAIProvider(providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey: os.Getenv("OPENAI_API_KEY"),
        Model:  "gpt-4o",  // 高质量内容生成
    },
}, logger)

// 工具阶段使用便宜模型
toolProvider := openai.NewOpenAIProvider(providers.OpenAIConfig{
    BaseProviderConfig: providers.BaseProviderConfig{
        APIKey: os.Getenv("OPENAI_API_KEY"),
        Model:  "gpt-4o-mini",  // 工具调用阶段
    },
}, logger)
```

### 服务器配置

```yaml
# config.yaml
server:
  environment: production     # development / production
  bind_address: ":8080"       # HTTP 监听地址
  metrics_bind_address: "127.0.0.1:9090"  # 指标端点（默认仅本地）
  enable_pprof: false         # 生产环境禁用 pprof
  allow_no_auth: false        # 生产环境必须认证
  rate_limit_rps: 100         # 全局限流 RPS
  rate_limit_burst: 200       # 突发上限
```

> ⚠️ **安全警告**: `allow_no_auth: true` 仅允许在 development 环境使用，production 环境启动时会强制拒绝。

### 工作流检查点配置

```yaml
workflow_checkpoint:
  enabled: true
  backend: redis              # file / redis / postgres
  ttl: 24h                    # 检查点保留时间
  redis:
    addr: "localhost:6379"
    prefix: "workflow:checkpoint:"
```

### 预算与缓存配置

```yaml
# Token 预算管理
budget:
  enabled: true
  daily_limit: 1000000        # 每日 Token 限制
  alert_threshold: 0.8        # 80% 告警

# LLM 响应缓存
cache:
  enabled: true
  backend: redis
  ttl: 1h
  semantic_cache: true        # 启用语义缓存
```

## 🔍 验证安装

创建 `verify.go` 验证安装是否成功：

```go
package main

import (
    "context"
    "fmt"

    agentruntime "github.com/BaSui01/agentflow/agent/runtime"
    "github.com/BaSui01/agentflow/sdk"
    workflowruntime "github.com/BaSui01/agentflow/workflow/runtime"
)

func main() {
    _ = sdk.New(sdk.Options{})
    _ = agentruntime.DefaultBuildOptions()
    _ = workflowruntime.NewBuilder(nil, nil)
    _ = context.Background()

    fmt.Println("✅ sdk 统一入口加载成功")
    fmt.Println("✅ agent runtime 子模块加载成功")
    fmt.Println("✅ workflow runtime 子模块加载成功")
    fmt.Println("🎉 AgentFlow 安装验证通过！")
}
```

运行：
```bash
go run verify.go
```

## 🐛 常见安装问题

### 问题 1: Go 版本过低

```
error: module requires Go >= 1.24
```

**解决方案：**
```bash
# 检查 Go 版本
go version

# 升级 Go（macOS）
brew upgrade go

# 升级 Go（Linux）
sudo rm -rf /usr/local/go
wget https://go.dev/dl/go1.24.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.24.linux-amd64.tar.gz
```

### 问题 2: 网络问题导致下载失败

```
go: github.com/BaSui01/agentflow: connection refused
```

**解决方案：**
```bash
# 设置 Go 代理（中国大陆用户）
go env -w GOPROXY=https://goproxy.cn,direct

# 或使用其他代理
go env -w GOPROXY=https://goproxy.io,direct
```

### 问题 3: 依赖冲突

```
go: inconsistent vendoring
```

**解决方案：**
```bash
# 清理并重新下载依赖
go clean -modcache
go mod tidy
go mod download
```
