- [x] 唯一仓库级正式入口：`sdk.New(opts).Build(ctx)`
- [x] 唯一单 Agent 正式入口：`agent/runtime`

- [x] 旧正式入口降级为内部构件或直接删除：`agent/runtime`、`agent/team`、`agent/team/internal/engines/multiagent`、`agent/adapters/teamadapter`

### 3.2 目标分层 / 分包

- [x] `sdk/`：仓库级统一装配入口
- [x] `agent/runtime/`：单 Agent runtime（Agent/Tool/Handoff/Session/RunConfig/RunEvent）
- [x] `agent/team/`：team facade + team contract + internal engines
- [x] `workflow/runtime/`：显式流程编排（Sequential / Parallel / Branch / Loop / Subgraph / HITL / Checkpoint）
- [x] `types/authz.go`：零依赖权限契约
- [x] `internal/usecase/authorization_service.go`：统一授权服务

### 3.3 依赖方向与主链约束

- [x] 不破坏现有层级依赖方向
- [x] 不绕过 `cmd -> bootstrap -> api -> domain` 启动主链
- [x] 不在 handler / adapter 层回灌业务实现
- [x] `workflow` 只依赖 `agent` contract，不吞并 `agent` runtime

### 3.4 根目录治理目标（如适用）

- [x] 顶层目录分类方案已定义（产品源码 / 正式文档 / 构建治理 / 示例验证 / 状态产物 / 外部参考）
- [x] `docs/architecture/` 作为活跃契约目录，`docs/archive/` 作为历史归档目录

### 3.5 外部参考框架对标（本轮硬切换的设计基线）

- [x] OpenAI Agents SDK：保留最小原语（Agent / Tool / Handoff / Session / Trace / Guardrail / MCP）
- [x] Google ADK：借鉴 workflow agents（Sequential / Parallel / Loop）和 Session / State / Memory
- [x] Microsoft Agent Framework：借鉴 Agent 与 Workflow 的硬边界、typed workflow、middleware、telemetry
- [x] LangGraph：借鉴 durable execution / checkpoint / HITL / graph orchestration / time travel
- [x] LangChain：借鉴 middleware、tool/runtime context、多 Agent handoff / subagent 模式
- [x] LlamaIndex / Haystack：借鉴 RAG-first agent、workflow event、pipeline/component 与 agentic RAG 的边界
- [x] PydanticAI：借鉴 typed output / deps / durable execution / validation retry
- [x] CrewAI：借鉴 Team 与 Flow 的职责分离
- [x] Mastra：借鉴 stateful agents / workflows / MCP / eval / observability 产品面
- [x] Dapr Agents：借鉴 durable workflow / actor / state store 基建抽象，不引入硬依赖
- [x] Agno / VoltAgent / BeeAI：借鉴 agents / teams / workflows / memory / control plane 的产品表面组织
- [x] smolagents / Pi：借鉴 coding-agent harness / code-as-action / session / RPC/headless 模式
- [x] CAMEL / MetaGPT：借鉴 role protocol / SOP / work product pipeline，但不照搬固定角色剧本
