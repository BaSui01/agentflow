## 1. 重构范围与目标

### 1.1 目标范围

- [x] 目标模块：`cmd/agentflow/`、`internal/app/bootstrap/`、`api/handlers/`、`internal/usecase/`
- [x] 关联 owner 模块：`llm/runtime/router/`、`agent/integration/hosted/`
- [x] 关联文档与守卫：`README.md`、`README_EN.md`、`docs/architecture/启动装配链路与组合根说明.md`、`docs/architecture/ADRs/001-分层架构设计.md`、`architecture_guard_test.go`、`scripts/arch_guard.ps1`
- [x] 关联活动计划：`docs/archive/agent-framework-legacy-2026-04/agent模块架构重构-2026-04-22.md`

### 1.2 业务 / 使用目标

- [x] 让调用方与维护者只面对一条正式启动 / 装配 / 热更新主链，不再需要在 `cmd` 与 `bootstrap` 间猜测“哪边是真正的 runtime 入口”
- [x] 让 `api/handlers` 真正只做协议适配，减少 adapter 层携带 infra 细节导致的演进成本
- [x] 让 `internal/usecase` 成为稳定的应用边界，使 handler 与 domain 间的契约变化可以在 usecase 层被吸收
- [x] 让仓库级架构口径可被 README / ADR / 守卫 / 脚本共同验证，而不是只在文档中“看起来正确”

### 1.3 架构目标

- [x] 唯一正式 serve-time 装配入口收口到 `internal/app/bootstrap.BuildServeHandlerSet(...)`
- [x] 唯一正式 text hot reload rebinding 入口收口到 `internal/app/bootstrap.ApplyReloadedTextRuntimeBindings(...)`
- [x] hosted tool / provider / approval 相关 handler bundle 入口收口到 `internal/app/bootstrap.BuildToolingHandlerBundle(...)`
- [x] `api/handlers` 删除 `*_store.go` 持久化实现，只保留 request decode / validation / auth context overwrite / error mapping / response serialization / stream writing
- [x] `internal/usecase` 为 chat / workflow 提供 usecase-owned DTO、stream/event 契约与 runtime ref，避免把 `llm/core`、`workflow/core` 具体类型继续暴露给 handler
- [x] `LLMProvider` / `LLMProviderAPIKey` 的 DB-backed store 归口到 `llm/runtime/router/`
- [x] `hosted.ToolProviderConfig` 的 DB-backed store 归口到 `agent/integration/hosted/`
- [x] `cmd/agentflow` 只保留生命周期、字段持有与调用 bootstrap 的职责，不再直接 new handler / service / workflow facade

### 1.4 非目标

- [x] 不在本轮重新设计 `agent`、`workflow`、`rag`、`llm` 的正式 public entry；既有正式入口口径保持不变
- [x] 不在本轮把内置 legacy DB-backed `MultiProviderRouter` 全面替换为 channel-routed 主链；本轮只治理边界与归口，不改产品路线
- [x] 不在本轮顺手清仓既有 lint 历史问题；仅处理本轮收口直接相关的问题
- [x] 不在本轮推翻 `docs/archive/agent-framework-legacy-2026-04/agent模块架构重构-2026-04-22.md`；若与本计划交叉，仅补充母计划与子计划衔接关系
