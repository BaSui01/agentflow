## 3. 目标架构

### 3.1 唯一正式入口

- [ ] 唯一正式入口：`agent.Builder`（对外通过 `agent/core/builder.go` 暴露）
- [ ] SDK 入口：`sdk/options.go` 继续作为 SDK 层的公开 API，内部调用 `agent.Builder`
- [ ] 旧正式入口降级或删除：
  - `agent/builder.go` → 迁移到 `agent/core/builder.go`
  - 任何 `NewXxxAgent` 函数 → 统一通过 `Builder` 模式

### 3.2 目标分层/分包

目标目录结构（7 层）：

```
agent/
├── core/                    # 核心层（唯一构造入口、Agent 接口）
│   ├── agent.go             # Agent 接口定义
│   ├── base.go              # 基础 Agent 实现
│   └── builder.go           # 唯一正式入口 Builder
│
├── capabilities/            # 能力层（可插拔能力）
│   ├── memory/              # 合并 memory + memorycore
│   ├── reasoning/           # 推理能力
│   ├── planning/            # 合并 planner + deliberation
│   ├── tools/               # 合并 skills + discovery
│   ├── guardrails/          # 合并 guardrails + guardcore
│   └── streaming/           # 流式能力
│
├── execution/               # 执行层
│   ├── runtime/             # 合并 runtime + execution + longrunning
│   ├── context/             # 上下文管理
│   ├── protocol/            # 协议处理
│   └── orchestration/       # 编排
│
├── collaboration/           # 协作层
│   ├── multiagent/          # 合并 multiagent + collaboration
│   ├── team/                # 合并 team + teamadapter + crews
│   ├── hierarchical/        # 层级模式
│   └── federation/          # 联邦模式
│
├── persistence/             # 持久化层
│   ├── checkpoint/          # 从根目录迁移 checkpoint_*.go
│   ├── conversation/        # 对话历史
│   └── artifacts/           # 产物存储
│
├── integration/             # 集成层（外部集成）
│   ├── deployment/          # 部署集成
│   ├── hosted/              # 托管服务
│   ├── k8s/                 # K8s 集成
│   ├── lsp/                 # LSP 集成
│   └── voice/               # 语音集成
│
├── observability/           # 可观测层
│   ├── monitoring/          # 原 observability 重命名
│   ├── evaluation/          # 评估
│   └── hitl/                # 人机交互
│
└── adapters/                # 适配层
    ├── chat.go              # 从 chat_request_adapter.go 迁移
    ├── declarative/         # 声明式配置
    ├── structured/          # 结构化输出
    └── handoff/             # 交接协议
```

- [ ] 创建 8 个顶层目录：core/, capabilities/, execution/, collaboration/, persistence/, integration/, observability/, adapters/
- [ ] 创建各层子目录（总计约 30 个子目录，从当前 38 个整合而来）

### 3.3 依赖方向与主链约束

- [ ] 依赖方向约束：
  - `core/` 只依赖 `types/` 和 `llm/`
  - `capabilities/` 可依赖 `core/`
  - `execution/` 可依赖 `core/` + `capabilities/`
  - `collaboration/` 可依赖 `core/` + `capabilities/` + `execution/`
  - `persistence/` 可依赖 `core/`
  - `integration/` 可依赖 `core/` + `capabilities/` + `execution/`
  - `observability/` 可依赖 `core/` + `capabilities/` + `execution/`
  - `adapters/` 可依赖 `core/` + `capabilities/`

- [ ] 不破坏现有层级依赖方向（agent 属于 Layer 2，不依赖 `cmd/`、`api/`）
- [ ] 不绕过 `cmd -> bootstrap -> api -> domain` 启动主链
- [ ] 不在 handler/adapter 层回灌业务实现

### 3.4 根目录治理目标（不适用）
