## 1. 当前收口事实

### 1.1 唯一正式入口

- 仓库级正式入口：`sdk.New(opts).Build(ctx)`
- `agent` 子模块正式 runtime 入口：`agent/execution/runtime.Builder`
- `agent/builder.go` 仅保留高级扩展和包内构建核心语义，不再作为正式 runtime 入口宣传

### 1.2 目录与职责

- root public surface 只保留 9 个非测试文件，负责公开类型、公共运行时表面和向下分层的薄门面
- `agent/adapters/chat.go` 是唯一正式 ChatRequest 构造边界
- `agent/collaboration/multiagent` 是多 Agent 协作正式实现归口
- `agent/collaboration/team` 是官方多 Agent facade
- `agent/persistence/checkpoint` 是 checkpoint store 实现归口
- `agent/capabilities/tools` 是技能/工具发现与技能管理的正式归口

### 1.3 本轮额外收口

- `examples/08_low_priority_features/main.go` 已从旧 `collaboration.*` 命名统一到 `multiagent.*`
- `scripts/livecheck/advanced_features.go` 已从旧 `skills.*` / `collaboration.*` 命名统一到 `tools.*` / `multiagent.*`
- `api/handlers/agent_test.go`、`api/handlers/handlers_extra_test.go`、`benchmarks/agent_concurrency_bench_test.go`、`e2e/agent_execution_test.go`、`e2e/agent_tool_sandbox_test.go`、`workflow/dsl/orchestration_integration_test.go` 已清理旧 `discovery` / `collaboration` / `execution` 残留
- 已新增迁移指南：[docs/migration/agent-refactor-2026-04-22.md](/mnt/e/code/agentflow/docs/migration/agent-refactor-2026-04-22.md)
