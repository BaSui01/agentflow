## 0. 背景

8 份重构文档均提到 `trace_id/run_id` 统一，但缺少一个跨层的统一字段规范。本文档定义全链路观测字段的唯一口径。

---

## 1. 统一观测字段定义

### 1.1 请求级字段（必填）

| 字段 | 类型 | 来源 | 语义 | 生命周期 |
|---|---|---|---|---|
| `trace_id` | `string` | API 入口生成（或透传外部） | 单次用户请求的全链路追踪标识 | 请求级 |
| `run_id` | `string` | Agent/Workflow 执行入口生成 | 单次执行运行标识 | 运行级 |
| `parent_run_id` | `string` | 子代理 spawn 时注入 | 父运行标识（多 Agent 场景） | 运行级 |
| `tenant_id` | `string` | API 入口注入 | 租户标识（多租户场景） | 请求级 |
| `user_id` | `string` | API 入口注入 | 用户标识 | 请求级 |

### 1.2 节点级字段（按需填充）

| 字段 | 类型 | 来源 | 语义 | 适用层 |
|---|---|---|---|---|
| `workflow_id` | `string` | Workflow 层 | 工作流定义标识 | workflow |
| `node_id` | `string` | Workflow/Agent 步骤 | 当前执行节点标识 | workflow/agent |
| `step_type` | `string` | 步骤实现 | 步骤类型（`llm/tool/human/code/agent`） | workflow |
| `capability` | `string` | LLM Gateway | 能力类型（`chat/image/video/embedding/...`） | llm |
| `provider` | `string` | LLM Gateway 路由决策 | 供应商标识 | llm |
| `model` | `string` | LLM Gateway 路由决策 | 模型标识 | llm |
| `route_role` | `string` | LLM Gateway | 路由角色（`primary/tool`） | llm |

### 1.3 度量字段（出口统一记录）

| 字段 | 类型 | 来源 | 语义 |
|---|---|---|---|
| `latency_ms` | `int64` | 各层出口 | 本层处理耗时（毫秒） |
| `prompt_tokens` | `int` | LLM Gateway 出口 | 输入 token 数 |
| `completion_tokens` | `int` | LLM Gateway 出口 | 输出 token 数 |
| `total_tokens` | `int` | LLM Gateway 出口 | 总 token 数 |
| `cost` | `float64` | LLM Gateway 出口 | 本次调用成本 |
| `retrieval_latency_ms` | `int64` | RAG 出口 | 检索耗时 |
| `rerank_latency_ms` | `int64` | RAG 出口 | 重排耗时 |
| `topk` | `int` | RAG 出口 | 检索返回条数 |
| `hit_count` | `int` | RAG 出口 | 命中文档数 |
| `context_tokens` | `int` | RAG 出口 | 上下文 token 数 |

## 2. 透传规则

```text
api (生成 trace_id, 注入 tenant_id/user_id)
  -> workflow (生成 run_id, 附加 workflow_id/node_id)
    -> agent (继承 trace_id/run_id, 子代理生成独立 run_id + parent_run_id)
    -> rag (继承 trace_id/run_id, 附加 retrieval 度量)
      -> llm/gateway (继承全部上下文, 附加 provider/model/capability/usage/cost)
```

- `trace_id` 全链路透传，不允许中途重新生成。
- `run_id` 在 Agent/Workflow 执行入口生成，子代理生成独立 `run_id` 并设置 `parent_run_id`。
- 度量字段仅在各层出口记录，不在中间环节重复累计。
- `types/context.go` 的 `With*/From*` 函数是唯一的上下文注入/读取入口。
