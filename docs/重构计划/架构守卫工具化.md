## 1. 业界工具对比

| 工具 | 语言 | 配置方式 | 核心能力 | CI 集成 | 维护成本 |
|---|---|---|---|---|---|
| **go-arch-lint** | Go | YAML (`.go-arch-lint.yml`) | 组件定义 + `mayDependOn` 规则 | CLI + CI | 低（声明式） |
| **arch-go** | Go | YAML (`arch-go.yml`) | 依赖规则 + 命名规则 + 循环检测 | `go test` 集成 | 低（声明式） |
| **depguard** | Go | golangci-lint 配置 | 包级 allow/deny 列表 | golangci-lint | 低 |
| 手写 test + ps1 | Go/PS | 代码 | 完全自定义 | 自定义 | 中高 |

## 2. 建议方案

推荐 **go-arch-lint + 保留现有 test 中的自定义检查**（如文件预算）。

理由：
- go-arch-lint 的 YAML 声明式配置比手写 test 更可读、更易维护。
- 新增依赖规则只需改 YAML，不需要写 Go 代码。
- 文件预算等自定义检查保留在 `architecture_guard_test.go`（go-arch-lint 不支持此类规则）。

示例配置（`.go-arch-lint.yml`）：

```yaml
version: 3
workdir: .
components:
  types:       { in: types/** }
  llm_core:    { in: llm/core/** }
  llm_gateway: { in: llm/gateway/** }
  llm_runtime: { in: llm/runtime/** }
  llm_caps:    { in: llm/capabilities/** }
  llm_provs:   { in: llm/providers/** }
  agent:       { in: agent/** }
  rag:         { in: rag/** }
  workflow:    { in: workflow/** }
  api:         { in: api/** }
  cmd:         { in: cmd/** }
  internal:    { in: internal/** }

commonComponents:
  - types

deps:
  types:
    mayDependOn: []
  llm_core:
    mayDependOn: [types]
  llm_gateway:
    mayDependOn: [types, llm_core, llm_runtime, llm_caps]
  llm_runtime:
    mayDependOn: [types, llm_core]
  llm_caps:
    mayDependOn: [types, llm_core, llm_provs]
  llm_provs:
    mayDependOn: [types, llm_core]
  agent:
    mayDependOn: [types, llm_core, llm_gateway, rag]
  rag:
    mayDependOn: [types, llm_core, llm_caps]
  workflow:
    mayDependOn: [types, llm_core, llm_gateway, agent, rag]
  api:
    mayDependOn: [types, llm_core, llm_gateway, agent, rag, workflow]
  cmd:
    mayDependOn: [types, llm_core, llm_gateway, llm_runtime, llm_caps, llm_provs, agent, rag, workflow, api, internal]
```
