# 基础设施 SDK 统一入口重构计划（2026-04-17）

> 文档类型：可执行重构计划  
> 适用范围：`sdk/`、`agent/`、`workflow/`、`rag/`、`llm/`、相关启动装配/文档/示例  
> 核心约束：每个子模块只能保留一个正式入口；删除旧入口；不保留兼容分支；不保留双实现  
> 迁移策略：单轨替换，先补缺失 runtime builder，再删除 legacy public surface，再统一文档与守卫  
> 并行策略：`WS-1 workflow+sdk`、`WS-2 agent` 可并行；`WS-3 docs/guards` 在代码主链稳定后串行收尾

## 1. 重构目标

### 1.1 业务目标

- [x] 外部项目导入当前仓库时，只认 `sdk.New(opts).Build(ctx)` 一个正式入口
- [ ] `sdk` 不再同时提供顶层 legacy 直注入口（`Provider/ToolProvider/Ledger`）
- [ ] `agent` 不再存在 `runtime.Builder / NewAgentBuilder / CreateAgent / NewBaseAgent` 并列为主路径
- [ ] `workflow` 不再依赖外层手工拼装 `DAGExecutor + Facade + Parser`
- [ ] README / docs / examples 不再并列宣传旧入口

### 1.2 架构目标

- [x] 仓库对外唯一正式入口定义为 `sdk.New(opts).Build(ctx)`
- [x] `agent` 子模块唯一 runtime 入口定义为 `agent/runtime.Builder`
- [x] `rag` 子模块唯一 runtime 入口定义为 `rag/runtime.Builder`
- [x] `workflow` 执行唯一入口定义为 `workflow.Facade.ExecuteDAG(...)`
- [ ] `workflow` 补齐唯一 runtime 装配入口 `workflow/runtime.Builder`
- [x] `llm` 对上层唯一运行入口定义为 `compose.Runtime.Gateway`

### 1.3 允许的唯一入口

- [x] 仓库级正式入口：`sdk.New(opts).Build(ctx)`
- [x] `agent` runtime 入口：`agent/runtime.NewBuilder(...).Build(ctx, cfg)`
- [x] `rag` runtime 入口：`rag/runtime.NewBuilder(...)`
- [x] `workflow` 执行入口：`workflow.Facade.ExecuteDAG(...)`
- [x] `llm` runtime 装配入口：`llm/runtime/compose.Build(...)`
- [x] `llm` routed provider 装配入口：`llm/runtime/router.BuildChannelRoutedProvider(...)`
- [x] `llm` 配置驱动 provider 构造入口：`llm/providers/vendor.NewChatProviderFromConfig(...)`
