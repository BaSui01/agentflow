## 💡 系统性改进方案

### 短期修复（直接修复具体问题）
- [ ] 修复 `developer` 角色准入不一致（V-001）。
- [ ] 明确并统一 `types.Message` 与 `api.Message` 的字段契约（C-001, C-002）。
- [ ] 修复 `WithRoles/Roles` 的切片可变引用问题（T-001）。

### 中期防护（自动化检查机制）
- [ ] 在架构守卫中新增“DTO 映射完整性检查”（至少覆盖关键字段）。
- [ ] 增加“types 层未使用符号检查”与“配置模型单轨检查”。

### 长期治理（架构层面提升）
- [ ] 统一 `types` 层定位：纯数据契约层 vs 低层行为契约层，二选一并更新 ADR。
- [ ] 对大重构采用“单轨迁移”：新配置模型生效即删除旧模型路径。
