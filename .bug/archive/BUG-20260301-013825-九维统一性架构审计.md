## 🔥 交叉分析：跨维度关联攻击链

### 攻击链 1：输入 → 错误 → 观测 → 安全（四连击）
```
V-001 分页无上限 → E-010 WriteJSON 错误被吞 → O-004 日志无 request_id → X-007 SSRF 无法追踪
```
攻击者发送 `limit=999999` → 服务器 OOM → 错误被吞无日志 → 无法溯源攻击者

### 攻击链 2：状态 → 并发 → 性能（三连击）
```
S-001 状态定义重复 → T-001 并发修改 nodeResults → P-003 Redis Keys() 阻塞
```
不同包的状态定义不一致 → 并行执行时状态判断错误 → 触发全表扫描清理

### 攻击链 3：契约 → 错误 → 安全（三连击）
```
C-008 StreamChunk Error 未转换 → E-008 json.Marshal 错误被忽略 → X-005 API Keys 可能暴露
```
流式错误丢失 → 错误响应格式混乱 → 敏感信息可能泄露到客户端

### 攻击链 4：输入 → 并发 → 崩溃（三连击）
```
V-015 Nodes 数量无上限 → T-001 并发修改 map → E-007 goroutine 无 panic recovery
```
恶意 DSL 包含 10000+ 节点 → 并行执行触发 map 竞态 → panic 无 recovery 导致进程崩溃
